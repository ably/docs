Use the `ably apps rules create` command to create a new [channel rule](/docs/channels#rules) for an Ably application. Channel rules define behavior for channels matching a specific namespace.

## Synopsis

<Code>
```shell
ably apps rules create --name <name> [flags]
```
</Code>

## Flags

### `--name` <a id="name"/> <RequiredBadge />

The name of the channel rule to create. The name defines the namespace that matching channels belong to.

### `--app` <a id="app"/>

The app ID to create the rule for. Uses the currently selected app if not specified.

### `--persisted` <a id="persisted"/>

Enable message persistence for matching channels. Defaults to `false`.

### `--push-enabled` <a id="push-enabled"/>

Enable push notifications for matching channels. Defaults to `false`.

### `--authenticated` <a id="authenticated"/>

Require authentication for operations on matching channels.

### `--tls-only` <a id="tls-only"/>

Restrict matching channels to TLS connections only.

### `--persist-last` <a id="persist-last"/>

Persist only the last message on matching channels.

### `--populate-channel-registry` <a id="populate-channel-registry"/>

Populate the channel registry for matching channels, enabling channel enumeration.

### `--mutable-messages` <a id="mutable-messages"/>

Enable message update and delete functionality on matching channels. Enabling this flag automatically enables message persistence.

### `--batching-enabled` <a id="batching-enabled"/>

Enable message batching for matching channels.

### `--batching-interval` <a id="batching-interval"/>

The batching interval in milliseconds for matching channels. Only applicable when `--batching-enabled` is set.

### `--conflation-enabled` <a id="conflation-enabled"/>

Enable message conflation for matching channels.

### `--conflation-interval` <a id="conflation-interval"/>

The conflation interval in milliseconds for matching channels. Only applicable when `--conflation-enabled` is set.

### `--conflation-key` <a id="conflation-key"/>

The conflation key for matching channels. Only applicable when `--conflation-enabled` is set.

## Global flags

* [`--json`](/docs/cli#json) or [`--pretty-json`](/docs/cli#pretty-json) — JSON output. Mutually exclusive flags.
* [`--verbose`](/docs/cli#verbose) — Enable verbose logging. Can be combined with `--json` or `--pretty-json`.

## Examples

Create a channel rule with message persistence enabled:

<Code>
```shell
ably apps rules create --name "chat" --persisted
```
</Code>

Create a channel rule with mutable messages enabled:

<Code>
```shell
ably apps rules create --name "updates" --mutable-messages
```
</Code>

Create a channel rule with push notifications enabled:

<Code>
```shell
ably apps rules create --name "alerts" --push-enabled
```
</Code>

Create a channel rule with persistence and push notifications for a specific app:

<Code>
```shell
ably apps rules create --name "notifications" --persisted --push-enabled --app "My App"
```
</Code>

Create a channel rule and output the result in JSON format:

<Code>
```shell
ably apps rules create --name "chat" --persisted --json
```
</Code>
