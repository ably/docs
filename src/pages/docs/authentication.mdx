All Pub/Sub clients must authenticate with Ably before they can publish or subscribe to channels. Use [token authentication](/docs/platform/auth/token) for client-side applications, and [basic authentication](/docs/platform/auth/basic) for trusted server environments.

## Authentication flow

The recommended authentication flow for client-side applications is:

1. Your auth server authenticates the user.
2. Your auth server issues an Ably-compatible token (JWT format is recommended for most apps).
3. The client SDK fetches tokens with `authCallback` and refreshes them automatically before expiry.

<Aside data-type='important'>
Never use API keys in client-side code. API keys do not expire and are not suitable for untrusted clients.
</Aside>

## Server setup

Create an endpoint that validates user-provided credentials and returns JWTs with the appropriate capabilities:

<Code>
```javascript
// Server-side JWT with channel-scoped capabilities
import jwt from 'jsonwebtoken';

const [keyName, keySecret] = process.env.ABLY_API_KEY.split(':');

const ablyJwt = jwt.sign(
  {
    'x-ably-capability': JSON.stringify({
      'your-channel': ['publish', 'subscribe', 'presence', 'history'],
    }),
    'x-ably-clientId': userId,
  },
  keySecret,
  { algorithm: 'HS256', keyid: keyName, expiresIn: '1h' }
);
```

## Client setup

Use `authCallback` to fetch tokens from your auth server. The SDK handles token refresh automatically:

<Code>
```javascript
import * as Ably from 'ably';

const realtime = new Ably.Realtime({
  authCallback: async (tokenParams, callback) => {
    try {
      const response = await fetch('/api/ably-token', { credentials: 'include' });
      if (!response.ok) throw new Error('Auth failed');
      const jwt = await response.text();
      callback(null, jwt);
    } catch (error) {
      callback(error, null);
    }
  },
});
```

```react

import * as Ably from 'ably';
import { AblyProvider } from 'ably/react';

export function App() {
  const client = useMemo(
    () =>

            const response = await fetch('/api/ably-token', { credentials: 'include' });
            if (!response.ok) throw new Error('Auth failed');
            callback(null, await response.text());
          } catch (error) {
            callback(error, null);
          }
        },
      }),
    []
  );

  return (
    <AblyProvider client={client}>
      <YourApp />
    </AblyProvider>
  );
}
```

## Pub/Sub capabilities

Capabilities are permissions that control what operations a client can perform. When you create a token, you specify which capabilities the client has:

| Capability | Description |
|------------|-------------|
| `publish` | Publish messages to channels |
| `subscribe` | Subscribe to messages, presence state changes, and annotations on channels |
| `presence` | Register presence on a channel (enter, update, and leave) |
| `history` | Retrieve message and presence history on channels |
| `annotation-publish` | Publish [annotations](/docs/messages/annotations) to messages on a channel |
| `annotation-subscribe` | Subscribe to individual [annotations](/docs/messages/annotations) on a channel |
| `message-update-own` | [Update messages](/docs/messages/updates-deletes) where the original publisher's `clientId` matches the updater's `clientId` |
| `message-update-any` | [Update any message](/docs/messages/updates-deletes) on the channel |
| `message-delete-own` | [Delete messages](/docs/messages/updates-deletes) where the original publisher's `clientId` matches the deleter's `clientId` |
| `message-delete-any` | [Delete any message](/docs/messages/updates-deletes) on the channel |
| `channel-metadata` | Get metadata for a channel and enumerate channels |
| `push-subscribe` | Subscribe devices for push notifications on channels |
| `push-admin` | Manage device registrations and push subscriptions for all devices in an app |
| `privileged-headers` | Set data in the privileged section of [message extras](/docs/api/realtime-sdk/messages#extras) |

## Channel-scoped capabilities

You can scope capabilities to specific channels, a namespace of channels, or all channels:

* `my-channel` - a specific channel
* `my-namespace:*` - all channels in the `my-namespace:` namespace
* `*` - all channels

For example, to grant different permissions on different channels:

<Code>
```javascript
'x-ably-capability': JSON.stringify({
  'public:*': ['subscribe'],
  'private:user-123': ['publish', 'subscribe', 'presence'],
})
```
</Code>

## Token lifecycle and permission updates

- With `authCallback` or `authUrl`, token refresh is automatic and handled by the SDK.
- To change a user's capabilities during an active session, issue a new token from your auth server and re-authenticate:

<Code>
```javascript
// Re-authenticate to pick up updated capabilities
await realtime.auth.authorize();
```
</Code>

- To immediately remove access, [revoke issued tokens](/docs/platform/auth/revocation).
- If your capability JSON is too large for JWT or must remain confidential, use native [Ably Tokens](/docs/platform/auth/token/ably-tokens).
