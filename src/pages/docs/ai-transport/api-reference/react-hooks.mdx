AI Transport provides React hooks for building chat UIs. Generic hooks are available from `@ably/ai-transport/react` and work with any codec. Vercel-specific hooks are available from `@ably/ai-transport/vercel/react`.

Hooks are context readers. The transport is created inside a `TransportProvider` (generic) or `ChatTransportProvider` (Vercel) higher in the tree; the hooks resolve it through context. Pass `channelName` to disambiguate when multiple providers are nested.

## Generic hooks <a id="generic"/>

Import from the React entry point:

<Code>
```javascript
import {
  TransportProvider,
  useClientTransport,
  useView,
  useCreateView,
  useTree,
  useActiveTurns,
  useAblyMessages,
} from '@ably/ai-transport/react';
```
</Code>

### TransportProvider <a id="transport-provider"/>

Creates a `ClientTransport` and makes it available to descendants. Wraps children with Ably's `ChannelProvider` using `channelName`. Descendants resolve the transport with `useClientTransport`.

<Code>
```javascript
<TransportProvider channelName="ai:demo" codec={UIMessageCodec}>
  <Chat />
</TransportProvider>
```
</Code>

| Prop | Type | Description |
| --- | --- | --- |
| channelName | `string` | The Ably channel name to subscribe to. Also the context registry key. |
| codec | `Codec<TEvent, TMessage>` | Codec for encoding and decoding messages on this channel. |
| clientId, headers, body, credentials, fetch, messages, logger, api | — | All other `ClientTransportOptions` except `channel` (managed internally). `api` is required on the generic entry point; the Vercel entry defaults it to `/api/chat`. |

### useClientTransport <a id="use-client-transport"/>

Read a `ClientTransport` from the nearest `TransportProvider`. The hook does not create or own the transport.

<Code>
```javascript
function useClientTransport(options?: {
  channelName?: string,
  skip?: boolean,
  onError?: (error: Ably.ErrorInfo) => void
}): ClientTransportHandle
```
</Code>

| Property | Type | Description |
| --- | --- | --- |
| channelName | `string` | The channel name passed to the enclosing `TransportProvider`. Omit to use the nearest provider. |
| skip | `boolean` | When `true`, return a stub transport that throws on any access. Useful when auth or another condition is not yet resolved. |
| onError | `(error) => void` | Called whenever the resolved transport emits an error event. The subscription is established once the transport resolves and removed on unmount. |

#### ClientTransportHandle <a id="client-transport-handle"/>

| Property | Type | Description |
| --- | --- | --- |
| transport | `ClientTransport` | The resolved transport. A throwing stub when `skip` is `true`, when no matching provider is found, or when transport construction failed. |
| transportError | `Ably.ErrorInfo \| undefined` | Set when no matching provider is found or transport construction failed and `skip` is `false`. `undefined` on success. |

Check `transportError` before using `transport` to avoid throws from the stub.

### useView <a id="use-view"/>

Subscribe to a view and return the visible node list with pagination, navigation, and write operations. When neither `transport` nor `view` is provided, the nearest `TransportProvider`'s transport is used.

<Code>
```javascript
function useView(options?: {
  transport?: ClientTransport | null,
  view?: View | null,
  limit?: number,
  skip?: boolean
}): ViewHandle
```
</Code>

| Property | Type | Description |
| --- | --- | --- |
| transport | `ClientTransport \| null` | Client transport whose default view to subscribe to. Defaults to the nearest provider when omitted. |
| view | `View \| null` | A specific view to subscribe to directly. Takes priority over `transport`. |
| limit | `number` | Maximum number of older messages to load per page. When provided, auto-loads the first page on mount. |
| skip | `boolean` | When `true`, skip all subscriptions and return an empty handle. |

#### ViewHandle <a id="view-handle"/>

| Property | Type | Description |
| --- | --- | --- |
| messages | `TMessage[]` | The visible domain messages along the selected branch. |
| nodes | `MessageNode<TMessage>[]` | Visible conversation nodes along the selected branch. |
| send | `(messages, options?) => Promise<ActiveTurn>` | Send one or more user messages on this view's selected branch. |
| regenerate | `(messageId, options?) => Promise<ActiveTurn>` | Regenerate an assistant message, using this view's branch for history. |
| edit | `(messageId, newMessages, options?) => Promise<ActiveTurn>` | Edit a user message, forking from this view's branch. |
| update | `(msgId, events, options?) => Promise<ActiveTurn>` | Amend an existing message and start a continuation turn (for example, tool results). |
| hasOlder | `boolean` | Whether older messages can be revealed via `loadOlder`. |
| loadOlder | `() => Promise<void>` | Load older messages from history. No-op if already loading. |
| loading | `boolean` | Whether a page load is currently in progress. |
| loadError | `Ably.ErrorInfo \| undefined` | Set when the most recent `loadOlder` failed; cleared on the next successful load. |
| getSiblings | `(msgId) => TMessage[]` | Get all sibling messages at a fork point. |
| hasSiblings | `(msgId) => boolean` | Whether a message has sibling alternatives. |
| getSelectedIndex | `(msgId) => number` | Index of the currently selected sibling at a fork point. |
| select | `(msgId, index) => void` | Select a sibling at a fork point by index. Triggers a view update. |
| getNode | `(msgId) => MessageNode \| undefined` | Get a node by message ID, or undefined if not found. |

The write operations (`send`, `regenerate`, `edit`, `update`) used to be standalone hooks (`useSend`, `useEdit`, `useRegenerate`). They are now methods on the `ViewHandle` returned by `useView`. Destructure them out as needed.

### useCreateView <a id="use-create-view"/>

Create an independent view over the same conversation tree and subscribe to it. Each created view has its own branch selection and pagination state. Returns the same `ViewHandle` as `useView`.

<Code>
```javascript
function useCreateView(options?: {
  transport?: ClientTransport | null,
  limit?: number,
  skip?: boolean
}): ViewHandle
```
</Code>

The created view is closed on unmount or when the transport changes.

### useTree <a id="use-tree"/>

Provide stable structural query callbacks backed by the transport's conversation tree. The hook does not re-render on tree changes; use `useView` for reactive updates.

<Code>
```javascript
function useTree(options?: {
  transport?: ClientTransport
}): TreeHandle
```
</Code>

#### TreeHandle <a id="tree-handle"/>

| Property | Type | Description |
| --- | --- | --- |
| getSiblings | `(msgId) => TMessage[]` | Get all sibling messages at a fork point. |
| hasSiblings | `(msgId) => boolean` | Whether a message has sibling alternatives. |
| getNode | `(msgId) => MessageNode \| undefined` | Get a node by message ID. |
| getHeaders | `(msgId) => Record<string, string> \| undefined` | Get the stored transport headers for a message. |

### useActiveTurns <a id="use-active-turns"/>

Subscribe to active turns on the channel. Returns a reactive `Map` keyed by client ID, where each value is a `Set` of turn IDs for that client. Re-renders when turns start or end.

<Code>
```javascript
function useActiveTurns(options?: {
  transport?: ClientTransport | null
}): Map<string, Set<string>>
```
</Code>

Use `.size` to check if any turns are active, `.has(clientId)` to check a specific client, and `.get(clientId)` to retrieve the set of turn IDs. See [concurrent turns](/docs/ai-transport/features/concurrent-turns) for usage examples.

### useAblyMessages <a id="use-ably-messages"/>

Subscribe to raw Ably messages on the transport's channel. Useful for building custom message handling outside the view abstraction.

<Code>
```javascript
function useAblyMessages(options?: {
  transport?: ClientTransport,
  skip?: boolean
}): Ably.InboundMessage[]
```
</Code>

## Vercel hooks <a id="vercel"/>

Import from the Vercel React entry point:

<Code>
```javascript
import {
  ChatTransportProvider,
  useChatTransport,
  useMessageSync,
} from '@ably/ai-transport/vercel/react';
```
</Code>

The Vercel React entry also re-exports `TransportProvider`, `useClientTransport`, `useView`, `useCreateView`, `useTree`, `useActiveTurns`, and `useAblyMessages` with `UIMessageChunk`/`UIMessage` type parameters pre-bound, so call sites do not need explicit type arguments.

### ChatTransportProvider <a id="chat-transport-provider"/>

Creates a `ChatTransport` adapter for Vercel's `useChat`, alongside the underlying `ClientTransport`. Wraps children with `TransportProvider` (using `UIMessageCodec`), so the Ably channel lifecycle is managed in one place.

<Code>
```javascript
<ChatTransportProvider channelName="ai:demo">
  <Chat />
</ChatTransportProvider>
```
</Code>

| Prop | Type | Description |
| --- | --- | --- |
| channelName | `string` | The Ably channel name to subscribe to. Also the context registry key. |
| api, clientId, headers, body, credentials, fetch, messages, logger | — | All other `VercelClientTransportOptions`. `api` defaults to `/api/chat`. |

### useChatTransport <a id="use-chat-transport"/>

Read a `ChatTransport` and its underlying `ClientTransport` from the nearest `ChatTransportProvider`. The hook is a context reader; it does not create transport state.

<Code>
```javascript
function useChatTransport(options?: {
  channelName?: string,
  skip?: boolean
}): ChatTransportHandle
```
</Code>

| Property | Type | Description |
| --- | --- | --- |
| channelName | `string` | The channel name passed to the enclosing `ChatTransportProvider`. Omit to use the nearest provider. |
| skip | `boolean` | When `true`, return stub transports that throw on any access. |

#### ChatTransportHandle <a id="chat-transport-handle"/>

| Property | Type | Description |
| --- | --- | --- |
| transport | `ClientTransport<UIMessageChunk, UIMessage>` | The underlying client transport, also available via `useClientTransport`. |
| chatTransport | `ChatTransport` | The Vercel-compatible adapter for use with `useChat`. |
| transportError | `Ably.ErrorInfo \| undefined` | Set when no matching `TransportProvider` was found or transport construction failed. |
| chatTransportError | `Ably.ErrorInfo \| undefined` | Set when no matching `ChatTransportProvider` was found or transport construction failed. |

Typical usage:

<Code>
```javascript
const { chatTransport } = useChatTransport();
const { messages } = useChat({ transport: chatTransport });
```
</Code>

See [ChatTransportOptions](/docs/ai-transport/api-reference/vercel#chat-transport-options) for the available configuration when constructing the provider.

### useMessageSync <a id="use-message-sync"/>

Wire transport message updates into Vercel's `useChat` message state. Subscribes to the transport view's `update` event and replaces messages with the view's authoritative message list. Use this when you need Vercel's message array to reflect messages from other devices or browser tabs.

<Code>
```javascript
function useMessageSync(options: {
  setMessages: (updater: (prev: UIMessage[]) => UIMessage[]) => void,
  channelName?: string,
  skip?: boolean
}): void
```
</Code>

| Property | Type | Description |
| --- | --- | --- |
| setMessages | `(updater) => void` | The `setMessages` updater from `useChat()`. Required. |
| channelName | `string` | Channel name of the `ChatTransportProvider` to observe. Omit to use the nearest provider. |
| skip | `boolean` | When `true`, skip all subscriptions. |

<Code>
```javascript
const { chatTransport } = useChatTransport();
const { messages, setMessages } = useChat({ transport: chatTransport });
useMessageSync({ setMessages });
// messages now includes messages from all devices on this channel
```
</Code>
