Ably provides multiple authentication methods designed for different use cases and security requirements. This guide covers the implementation details for each method and when to use them.

## Key Authentication

Key authentication is the simplest method, using your Ably API key directly. It's designed for server-side applications where the API key can be securely stored.

### Implementation

<Code>
```javascript
const ably = new Ably.Realtime({
  key: 'your-app-id.key-id:key-secret'
});
```

```python
ably = AblyRealtime(key='your-app-id.key-id:key-secret')
```

```go
client, err := ably.NewRealtime(ably.WithKey("your-app-id.key-id:key-secret"))
```

```java
ClientOptions options = new ClientOptions();
options.key = "your-app-id.key-id:key-secret";
AblyRealtime ably = new AblyRealtime(options);
```

```php
$ably = new Ably\AblyRealtime(['key' => 'your-app-id.key-id:key-secret']);
```
</Code>

### When to Use Key Authentication

✅ **Recommended for:**
- Backend servers and services
- Server-side applications
- Trusted environments
- Microservices communication

❌ **Not recommended for:**
- Web browsers
- Mobile applications
- Client-side JavaScript
- Any untrusted environment

### Security Considerations

Key authentication requires the full API key, including the secret, which provides complete access to your Ably app. This creates several security concerns:

- **Full capabilities**: All operations permitted by the key are possible
- **No expiration**: Keys don't expire automatically
- **No revocation**: Individual connections cannot be revoked
- **Client identity**: Clients can claim any `clientId`

## Token Authentication

Token authentication uses short-lived tokens that are issued by your servers to client applications. This is the recommended approach for client-side authentication.

### Token Types

Ably supports two types of tokens:

#### 1. Ably Tokens

Ably Tokens are issued directly by the Ably service using your API key.

<Code>
```javascript
// Server-side: Create token request
const tokenRequest = await ably.auth.createTokenRequest({
    clientId: 'user-123',
    capability: { 'channel1': ['subscribe'], 'channel2': ['publish'] }
});

// Send tokenRequest to client
// Client-side: Use token request
const client = new Ably.Realtime({
    authUrl: '/api/ably/token-request'
});
```

```python
# Server-side: Create token request
token_request = await ably.auth.create_token_request({
    'client_id': 'user-123',
    'capability': {'channel1': ['subscribe'], 'channel2': ['publish']}
})

# Client-side: Use auth URL
client = AblyRealtime(auth_url='/api/ably/token-request')
```
</Code>

#### 2. JWT Tokens

JSON Web Tokens (JWT) provide a standardized way to securely transmit authentication information.

### Token Refresh Patterns

#### Using authUrl

The `authUrl` approach is recommended for web clients as it automatically handles token refresh.

<Code>
```javascript
const ably = new Ably.Realtime({
    authUrl: '/api/ably/token',
    authMethod: 'POST',
    authParams: { userId: '12345' },
    authHeaders: { 'Authorization': 'Bearer user-session-token' }
});
```

```swift
let options = ARTClientOptions()
options.authUrl = URL(string: "/api/ably/token")
options.authMethod = "POST"
options.authParams = ["userId": "12345"]
let ably = ARTRealtime(options: options)
```
</Code>

#### Using authCallback

For more control over the token acquisition process, use `authCallback`.

<Code>
```javascript
const ably = new Ably.Realtime({
    authCallback: async (tokenParams, callback) => {
        try {
            const response = await fetch('/api/ably/token', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify({
                    userId: getCurrentUserId(),
                    ...tokenParams
                })
            });

            if (!response.ok) throw new Error('Token request failed');

            const tokenRequest = await response.json();
            callback(null, tokenRequest);
        } catch (error) {
            callback(error, null);
        }
    }
});
```

```flutter
final options = ably.ClientOptions(
    authCallback: (ably.TokenParams tokenParams) async {
        final response = await http.post(
            Uri.parse('/api/ably/token'),
            body: jsonEncode({
                'userId': getCurrentUserId(),
                'capability': tokenParams.capability,
            }),
        );

        if (response.statusCode != 200) {
            throw Exception('Token request failed');
        }

        return ably.TokenRequest.fromJson(jsonDecode(response.body));
    }
);
```
</Code>

## JWT Authentication

JWT (JSON Web Tokens) provide a standardized, secure way to handle authentication with additional flexibility for custom claims.

### Creating Ably JWTs

Ably JWTs must be signed with your API key secret and include specific claims:

<Code>
```javascript
const jwt = require('jsonwebtoken');

function createAblyJWT(userId, capabilities) {
    const currentTime = Math.floor(Date.now() / 1000);

    const payload = {
        iss: 'your-app-id.key-id', // API key name
        sub: userId, // Subject (clientId)
        iat: currentTime, // Issued at
        exp: currentTime + 3600, // Expires in 1 hour
        'x-ably-capability': JSON.stringify(capabilities),
        'x-ably-clientId': userId
    };

    return jwt.sign(payload, 'your-key-secret', {
        algorithm: 'HS256',
        header: { typ: 'JWT', alg: 'HS256', kid: 'your-app-id.key-id' }
    });
}
```

```python
import jwt
import time

def create_ably_jwt(user_id, capabilities):
    current_time = int(time.time())

    payload = {
        'iss': 'your-app-id.key-id',  # API key name
        'sub': user_id,  # Subject (clientId)
        'iat': current_time,  # Issued at
        'exp': current_time + 3600,  # Expires in 1 hour
        'x-ably-capability': json.dumps(capabilities),
        'x-ably-clientId': user_id
    }

    headers = {
        'typ': 'JWT',
        'alg': 'HS256',
        'kid': 'your-app-id.key-id'
    }

    return jwt.encode(payload, 'your-key-secret', algorithm='HS256', headers=headers)
```

```go
import (
    "time"
    "github.com/golang-jwt/jwt/v4"
)

func createAblyJWT(userID string, capabilities map[string][]string) (string, error) {
    now := time.Now()

    claims := jwt.MapClaims{
        "iss": "your-app-id.key-id", // API key name
        "sub": userID,               // Subject (clientId)
        "iat": now.Unix(),           // Issued at
        "exp": now.Add(time.Hour).Unix(), // Expires in 1 hour
        "x-ably-capability": capabilities,
        "x-ably-clientId": userID,
    }

    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    token.Header["kid"] = "your-app-id.key-id"

    return token.SignedString([]byte("your-key-secret"))
}
```
</Code>

### Using JWTs with Ably

<Code>
```javascript
// Direct token usage
const ably = new Ably.Realtime({
    token: jwtToken,
    clientId: 'user-123'
});

// Or via auth callback that generates JWT
const ably = new Ably.Realtime({
    authCallback: async (tokenParams, callback) => {
        const jwt = await generateJWTFromYourServer(tokenParams);
        callback(null, jwt);
    }
});
```
</Code>

### Custom JWT Claims

JWTs can include custom claims for application-specific data:

<Code>
```javascript
const payload = {
    // Standard Ably claims
    iss: 'your-app-id.key-id',
    sub: 'user-123',
    'x-ably-capability': '{"*":["*"]}',
    'x-ably-clientId': 'user-123',

    // Custom claims for channel-specific permissions
    'ably.channel.private-user-123': 'owner',
    'ably.channel.public-*': 'participant',
    'ably.channel.*': 'guest',

    // Custom application data
    'userRole': 'premium',
    'features': ['analytics', 'priority-support']
};
```
</Code>

## Advanced Token Options

### Token Revocation Support

Enable token revocation for enhanced security:

<Code>
```javascript
// When creating tokens, enable revocation
const tokenRequest = await ably.auth.createTokenRequest({
    clientId: 'user-123',
    revocableTokens: true // Enable revocation for this token
});
```
</Code>

### Token TTL Management

Set appropriate token lifetimes based on your use case:

<Code>
```javascript
const tokenRequest = await ably.auth.createTokenRequest({
    clientId: 'user-123',
    ttl: 15 * 60 * 1000, // 15 minutes in milliseconds
    capability: { 'user-notifications': ['subscribe'] }
});
```
</Code>

### Handling Token Expiry

Implement graceful token renewal:

<Code>
```javascript
ably.auth.on('authorizationError', async (error) => {
    console.log('Token expired, requesting new token...');
    // SDK will automatically call authUrl or authCallback
});

// Monitor connection state changes
ably.connection.on('failed', (stateChange) => {
    if (stateChange.reason?.code === 40140) {
        console.log('Authentication failed - token may be invalid');
    }
});
```
</Code>

## Implementation Examples

### Server-Side Token Endpoint

<Code>
```javascript
// Express.js token endpoint
app.post('/api/ably/token', authenticate, async (req, res) => {
    try {
        const { userId } = req.user;
        const { capability } = req.body;

        // Validate user permissions
        const allowedCapability = await getUserCapabilities(userId);

        const tokenRequest = await ably.auth.createTokenRequest({
            clientId: userId,
            capability: intersectCapabilities(capability, allowedCapability),
            ttl: 30 * 60 * 1000 // 30 minutes
        });

        res.json(tokenRequest);
    } catch (error) {
        res.status(500).json({ error: error.message });
    }
});
```

```python
# Flask token endpoint
@app.route('/api/ably/token', methods=['POST'])
@authenticate
async def ably_token():
    try:

            'ttl': 30 * 60 * 1000  # 30 minutes
        })

        return jsonify(token_request)
    except Exception as e:
        return jsonify({'error': str(e)}), 500
```

## Best Practices

1. **Choose the right method**: Use key authentication for servers, token authentication for clients
2. **Implement automatic token refresh**: Use `authUrl` or `authCallback` for seamless renewal
3. **Set appropriate TTL**: Balance security (shorter TTL) with user experience
4. **Validate capabilities server-side**: Never trust client-provided capability requests
5. **Handle authentication errors gracefully**: Implement proper error handling and user feedback
6. **Use HTTPS**: Always encrypt authentication requests in production
7. **Monitor token usage**: Track token generation and usage patterns for security insights

## Troubleshooting

### Common Authentication Errors

- **40140 - Token authentication required**: Client is trying to connect without proper authentication
- **40141 - Token revoked**: The token has been explicitly revoked
- **40142 - Token expired**: The token has passed its expiry time
- **40143 - Invalid token**: The token format or signature is invalid
