To resolve issues effectively, refer to the error codes provided. Each error code is supported by a description of the problem, examples of when it might occur, and steps to address it.

If the provided information does not resolve your issue, contact Ably "support":https://ably.com/support. When contacting, include details such as your app ID, the error code, and any relevant logs to help troubleshoot. Include as much of the following information as possible:
* Provide timestamps in UTC format.
* Include complete SDK logs from the time of the failure. Ensure these logs show activity before and after the timeout, as SDKs retry failed requests by default.
* Specify which endpoints were accessed. Mention if you use custom client options, the environment setting, and the failing SDK operation.
* State the SDK/s you use, including the platform and versions.
* Include any stack traces related to the error.
* Indicate whether the issue occurs consistently or was a one-time event.
* Provide details to confirm whether the issue was related to your network or Ably's availability. For example, note whether other internet operations were succeeding simultaneously.
* Include the app ID associated with the request.

h2(#errors). Invalid input/request errors

h3(40000). 40000 - Bad request

Ably cannot understand the request due to a malformed request structure or other issues with the request format.

An example of this error is when a message is published while the connection is in a suspended state.

*Resolution:* Check the request format. Ensure the "connection":/error-handling#connections is in the connected state before publishing.

h3(#40001). 40001 - Invalid request body

The request contains invalid content, such as incorrect data types or missing required objects.

An example of this error is when a missing field, such as a @clientId@ for a token request.

*Resolution:* Review the "API references":/docs/api to ensure your request body matches the expected format.

h3(#40003). 40003 - Invalid parameter value

An invalid value was sent in the request.

An example of this error is when the TTL set for a token parameter exceeds the maximum allowed value.

*Resolution:* Identify the specific parameter causing the issue and resend the request with the corrected parameters.

h3(#40005). 40005 - Invalid credentials

The "API key":/auth#api-keys or token used to initialize the library is invalid.

An example of this error is using an invalid API key or token, such as passing an expired or incorrectly formatted token during initialization.

*Resolution:* Verify or create a valid "API key":/auth#api-keys with the required permissions.

h3(#40015). 40015 - Invalid device ID

This error occurs when the @id@ property in a @DeviceDetails@ object sent during a push notification registration request is invalid.

An example of this error is when an application sends a "push notification":/push#configure registration request with the following @DeviceDetails@ object:

```
“id”: 12345,
“platform”: “android”,
“device”: “phone”
```

The @id@ property is invalid because it is a numeric value instead of the required string.

*Resolution:*
- The @id@ property must be a string. Ensure the request includes a valid string value for the @id@ property, such as:

```
{
“id”: “12345”,
“platform”: “android”,
“device”: “phone”
}
```

h3(#40022). 40022 - Invalid resource

The requested resource is invalid or already exists.

An example of this error is attempting to create an app named "MyApp" when an app with that name already exists or attempting to create a user account with an email address that is already registered.

*Resolution:* Ensure unique app's name has valid capabilities in requests.

h2(#connection). Connection errors

h3(#80000). 80000 - Connection failed

The SDK is failing to connect to Ably, which is likely due to issues with client-side network connectivity.

An example of this error is a client application using the SDK fails to connect to Ably due to network issues, such as being offline or blocked by a firewall.

*Resolution:* No immediate action is required, as the SDK will automatically retry the "connection":/connect#connection-states after 15 seconds.

h3(#80002). 80002 - Connection suspended

The SDK cannot connect to Ably, likely due to client-side network issues, and has failed to establish a connection within 2 minutes.

An example of this error is the client application loses network connectivity, and after 2 minutes of failed retries, the connection transitions to the suspended state, indicating prolonged unavailability.

*Resolution:* Check your network connection and verify that the client device has stable internet access. Retry the "connection":/error-handling#connections and call the @.connect()@ method to reattempt a connection manually.

h3(#80003). 80003 - Generic disconnection error

The SDK has trouble connecting to Ably, likely due to client-side network connectivity issues. The specific cause cannot be determined.

An example of this error is a client application loses its connection to Ably due to an unspecified issue, such as Wi-Fi dropouts. The detailed logs or error messages do not explain the exact cause.

*Resolution:* No immediate action is required, as the SDK will automatically retry the connection after 15 seconds. Check your network connectivity and verify that the client device has stable internet access. Retry the connection and call the @.connect()@ method to reattempt a connection manually.

h3(#80018). 80018 - Invalid connection ID (invalid format)

An invalid connection ID was supplied.

An example of this error is an invalid connection ID is when the client application sends a message with a connection ID that doesn't match the current connection state due to an outdated SDK version.

*Resolution:* Upgrade to the latest version of the SDK to ensure that the connection lifecycle is properly managed and that valid connection IDs are used.

If you are seeing resumes failing with error 80018 “Invalid connection serial” in ably-js, this was a known bug in ably-js versions 1.2.30 through 1.2.33. Please upgrade to the latest version of ably-js to resolve this issue.

h2(#internal). Internal errors

h3(#500). 500/50000 - Internal error

There is an issue within Ably's system.

An example of this error is when a client attempts to publish a message to a channel, but the operation fails due to an unexpected server-side issue, such as a temporary outage or internal server misconfiguration.

*Resolution:* View Ably's status to check if there are any ongoing incidents or maintenance.

h3(#50001). 50001 - Internal channel error

There is an internal issue on an Ably channel.

An example of this error is when a client subscribes to a channel, but the subscription fails due to an unexpected server-side issue, such as a misconfigured channel state or a temporary backend error.

*Resolution:* View Ably's "status":https://status.ably.com/ to check if there are any ongoing incidents or maintenance.

h3(#50002). 50002 - Internal connection error

There is an internal connection issue.

An example of this error is when a client attempts to establish a connection with Ably, but the process fails due to a server-side connectivity issue, such as a misconfigured connection endpoint or a temporary outage affecting the connection layer.

*Resolution:* View Ably's "status":https://status.ably.com/ to check if there are any ongoing incidents or maintenance.

h3(#50003). 50003 - Timeout error

This error occurs when your request to Ably times out.

An example of this error is when a client tries to publish a message to a channel, but the operation times out without succeeding.

*Resolution:* View Ably's "status":https://status.ably.com/ to check if there are any ongoing incidents or maintenance.

h3(#50305). 50305 - Ably's routing layer was unable to service this request

Ably's routing layer cannot process a request due to an internal routing error.

An example of this error is when a client attempts to publish a message or retrieve data, but the operation fails due to an issue within Ably's routing infrastructure, such as an unexpected misconfiguration or temporary routing failure.

*Resolution:* View Ably's "status":https://status.ably.com/ to check if there are any ongoing incidents or maintenance.

h2(#channel). Channel errors

h3(#40011). 40011 - Stale ring state

When using the channel "enumeration":/api/rest-api#enumeration-rest API, a stale ring state occurs. This indicates that a pagination sequence has become invalid due to changes in the cluster state between successive calls.

An example of this error is when a client uses the channel enumeration API to paginate through a list of channels, but changes to the cluster state during the process cause the sequence to fail.

*Resolution:* Restart the enumeration process from the beginning to retrieve the complete result. Note that this issue does not affect enumerations satisfied within the first response page.

h3(#42912). 42912 - Channel iteration call already in progress

When performing concurrent metadata REST requests to retrieve a list of active channels, the API enforces a strict rate limit, allowing only one active request at any given time. Any additional concurrent requests will be rejected.

An example of this error is when an application makes multiple simultaneous REST requests to retrieve a list of active channels using the REST API. While one request is still in progress, a second request is initiated and is rejected due to the rate limit on concurrent calls.

*Resolution:* "Enumerate":/api/rest-api#enumeration-rest all channel requests. Ensure your application waits for the current channel iteration request to complete before initiating a new one.


h3(#90001). 90001 - Channel operation failed (invalid channel state)

This error occurs when you attempt to operate on a channel that is in a state that does not permit the operation.

An example of this error is when an application tries to "publish":/channels#publish a message to a channel that is in a failed state due to a prior error. As a result, the operation fails because actions cannot be performed on a channel in this state.

*Resolution:* Ensure the channel is in an appropriate state before performing any operations. Use the Ably SDK to monitor and handle channel "state":/channels#listen-for-state changes effectively.

h3(#90001). 90007 - Channel operation failed (invalid channel state)

This error occurs when you attempt to operate on a channel that is in a "state":/channels#states that does not permit the operation.

An example of this error is when an application tries to publish a message to a channel that is in a failed state due to a prior error. The operation fails because actions cannot be performed on a channel in this state.

*Resolution:* Ensure the channel is in an appropriate state before performing any operations. Use the Ably SDK to monitor channel state changes and handle transitions effectively.

h3(#90007). 90007 - Channel didn't attach within 10 seconds

This error occurs when a channel fails to attach within 10 seconds. Clients with poor internet connections are more likely to encounter this issue because the ACK response to an ATTACH request does not arrive within the default timeout period.

An example of this error is when a client reconnects after a disconnection and attempts to resume a channel's state. Messages sent during the period of disconnection have expired, and the system cannot guarantee continuity.

*Resolution:* Adjust the @realtimeRequestTimeout@ or @channelRetryTimeout@ to a higher value. Note that this error was incorrectly assigned to error code 91200 in older versions of the Ably Pub/Sub Java SDK.

h3(#91003). 91003 - Maximum member limit exceeded

This error occurs when the maximum member limit for a channel has been exceeded, preventing additional clients from joining.

An example of this error is An application attempting to add a new client to the "presence":/docs/presence-occupancy/presence?lang=javascript set of a channel, but the presence member limit for example, 100 members for the current subscription plan, has already been reached. The error prevents the additional client from joining.

*Resolution:* Use "batching":/channels#batch-publish to optimize how presence messages are sent and received. This can help reduce the number of individual presence updates on the channel. You can also check your channel "limits":g/pricing/limits to understand the presence limits for your current plan.

h2(#reactor). Reactor and ingress errors

h3(#70001-2). 70001/2 - Reactor operation failed (post operation returned unexpected code)

An integration rule fails because the endpoint returns an unexpected response code. There could also be an issue with your server connection. Also, if Ably sends a webhook event to your server and it repeatedly rejects the request, Ably retries several times. However, if the rejections continue, the operation will fail.

An example of this error is when an integration rule is configured to forward channel messages to a webhook endpoint, but the webhook responds with a status code of 500 instead of the expected 200, causing the integration to fail.

Another an example is when Ably publishes an event to your webhook endpoint, but your server responds with an error, such as a 4xx or 5xx HTTP status code.

*Resolution:* Verify that the endpoint used in the integration rule is configured correctly. Ensure the endpoint can handle the request as expected.

To resolve your webhook endpoint error, investigate your server's logs and configuration to identify the cause of the request rejections. Common issues include authentication failures or misconfigured endpoints. Ensure your server is properly set up to handle incoming webhook requests.

h3(#72000). 72000 - Ingress operation failed

This error occurs due to an internal issue with the ingress worker when it encounters an error while executing a rule.

An example of this error is when the ingress worker fails to process incoming data from an external source because the data format does not match the expected structure or contains invalid fields.

*Resolution:* The ingress worker is maintained by Ably, so you will need to raise a "support":https://ably.com/support ticket.

h3(#72002). 72002 - Ingress table is unhealthy

The ingress worker cannot access or retrieve data from the "outbox":/livesync/postgres#outbox-table or "nodes":/livesync/postgres#nodes-table table in the database as expected.

Examples of this error include misconfigurations in the database setup, inconsistencies between the provided configuration and the actual database schema or table names, or issues with database connectivity.

*Resolution:* Verify the configuration of the outbox and nodes tables in your database. Ensure they are set up correctly and match the rule definition.

h2(#push). Push notification errors

h3(#61002). 61002 - Activation failed: Present clientId is not compatible with existing device registration

This error occurs when you previously "activated":/configure/device#activate-devices a device for push notifications with a specific @clientId@, but then changed the @clientId@ used for authentication. The mismatch causes the error because the push notification setup tracks the @clientId@ and other details to prevent accidental changes between app launches. The @clientId@ is linked to registrations, such as subscribing by @clientId@.

An example of this error is when push notifications are activated with a @DeviceDetails@ object tied to a @clientId@ for a client device. After authenticating with a new @clientId@, an error occurs when the device tries to register again due to incompatible @DeviceDetails@ between the original and new @clientId@.

*Resolution:* If you need to change your client's @clientId@, deactivate and reactivate the device. This process triggers an internal @device.reset()@ call, which clears and resets the old device details.

h2(#spaces). Spaces and locking errors

h3(#101000). 101000 - Space name missing

The @spaces.get()@ method requires a name parameter to identify the "space":/spaces/space you want to create or interact with.

An example of this error is when a client attempts to call the @spaces.get()@ method without specifying a name parameter:

```javascript
const space = await spaces.get();

*Resolution:* Specify the name parameter when calling the @spaces.get()@ method:

```[realtime_javascript]
const space = await spaces.get('mySpace');
```

h3(#101001). 101001 - Not entered space

Some features require a member to "enter":/spaces/space#enter a space before they can be used.

An example of this error is when a member tries to lock a component but has not yet entered the space.

*Resolution:* Call the @space.enter()@ method before using any features. For example:

```[realtime_javascript]
const space = await spaces.get('mySpace');
space.enter({
  username: 'Claire Lemons',
  avatar: 'https://slides-internal.com/users/clemons.png',
});
```

h3(#101002). 101002 - Lock request exists

This error occurs when an existing "lock":/spaces/locking request is in a pending or locked state. The system does not support nested locks, so you must resolve the previous request before making a new lock request.

An example of this error is when a member attempts to acquire a lock on a cell, but a previous lock request for the same resource is still pending or in a locked state.

*Resolution:* Ensure the previous lock is released or the pending lock request is resolved before making a new one.

h3(#101003). 101003 - Lock is locked

This error occurs when the "lock":/spaces/locking remains locked, and the pending request fails to invalidate its status.

An example of this error is when a member tries to lock a document cell, but another member's lock request is still active and has not been released, preventing the new lock request from being processed.

*Resolution:* Ensure the current lock is released before attempting to acquire it again.Use the lock release mechanism provided in your application to explicitly release the lock, or wait for the lock to expire if an expiration time is configured.
