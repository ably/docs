Use the `ably apps update` command to update the configuration of an existing Ably application.

## Synopsis

<Code>
```shell
ably apps update <app-id> [flags]
```
</Code>

## Arguments

### `app-id` <a id="app-id"/> <RequiredBadge />

The ID of the application to update.

## Flags

### `--name` <a id="name"/>

The new name for the application.

### `--tls-only` <a id="tls-only"/>

Restrict the application to TLS connections only. When enabled, all connections to this app must use TLS encryption.

## Global flags

* [`--json`](/docs/cli#json) or [`--pretty-json`](/docs/cli#pretty-json) — JSON output. Mutually exclusive flags.
* [`--verbose`](/docs/cli#verbose) — Enable verbose logging. Can be combined with `--json` or `--pretty-json`.

## Examples

Update an application's name:

<Code>
```shell
ably apps update aBcDe1 --name "new-app-name"
```
</Code>

Restrict an application to TLS connections only:

<Code>
```shell
ably apps update aBcDe1 --tls-only
```
</Code>

Update an app and output the result in JSON format:

<Code>
```shell
ably apps update app-id --name "Updated App Name" --json
```
</Code>

Update an app using an access token environment variable:

<Code>
```shell
ABLY_ACCESS_TOKEN="YOUR_ACCESS_TOKEN" ably apps update app-id --name "Updated App Name"
```
</Code>
