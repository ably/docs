This reference covers every public API surface in the `@ably/ai-transport` package. The SDK is organised into four entry points so you only import what you need.

## Quick example <a id="quick-example"/>

A minimal client transport setup:

<Code>
```javascript
import * as Ably from 'ably';
import { createClientTransport } from '@ably/ai-transport/vercel';

const ably = new Ably.Realtime({ authUrl: '/auth' });
const channel = ably.channels.get('conversation-42');

const transport = createClientTransport({ channel });
```
</Code>

See the individual reference pages for the full API.

## API surfaces <a id="surfaces"/>

| Surface | Description | Reference |
| --- | --- | --- |
| ClientTransport | Subscribe to turns, build conversation views, cancel and control from the client. | [ClientTransport](/docs/ai-transport/api/javascript/client-transport) |
| ServerTransport | Manage turn lifecycle, stream responses, handle cancellation on the server. | [ServerTransport](/docs/ai-transport/api/javascript/server-transport) |
| Vercel AI SDK integration | Pre-built codec and factories for the Vercel AI SDK. | [Vercel AI SDK integration](/docs/ai-transport/api/javascript/vercel) |
| Codec | Interface for bridging any AI framework to Ably messages. | [Codec](/docs/ai-transport/api/javascript/codec) |
| Errors | The `ErrorInfo` type and numeric error codes with recovery guidance. | [Errors](/docs/ai-transport/api/errors) |
| React | Providers and hooks for building chat UIs. | [Providers](/docs/ai-transport/api/react/providers) and the per-hook pages under `react/`. |

## Entry points <a id="entry-points"/>

| Entry point | Import path | Contents |
| --- | --- | --- |
| Core | `@ably/ai-transport` | `createClientTransport`, `createServerTransport`, codec interfaces |
| React | `@ably/ai-transport/react` | Generic React hooks (`useClientTransport`, `useView`, and so on) |
| Vercel | `@ably/ai-transport/vercel` | `UIMessageCodec`, Vercel-specific transport factories |
| Vercel React | `@ably/ai-transport/vercel/react` | `useChatTransport`, `useMessageSync` |

<Aside data-type='note'>
The Vercel entry points re-export the core transport factories with the codec pre-bound. If you use `@ably/ai-transport/vercel`, you do not need to import from `@ably/ai-transport` separately.
</Aside>
