h2. App statistics

Monitor and analyze your app's performance through a statistics table and chart:

* View last realtime data to quickly spot app issues.
* Track messages, presence events, and performance metrics to understand usage and capacity.
* Compare data over time to identify trends and optimize resources.
* Adjust chart views with zoom options and set specific time ranges for detailed insights.

See: "App statistics.":/account/dashboard/stats

h2. Getting started

Connect to Ably, listen for a message, send a message, and close the connection:

* Obtain an API key to authenticate with Ably and gain full access to all channels.
* Install the Ably SDK.
* Create a file, connect to Ably, subscribe to a channel, publish messages, and close the connection using the provided JavaScript code.
* Run your script, which connects, publishes a message, subscribes to the channel, and closes after a set timer.

See: "Getting started.":/getting-started/quickstart

h2. App API keys

Create, manage, and configure API keys by setting capabilities:

* Users can create a new API key, assigning a descriptive name for easy identification.
* Capabilities include permissions like publishing messages, subscribing to channels, retrieving message history, managing presence, and accessing statistics.
* Control access by defining whether the key has unrestricted access, channel-only, queue-only, or specific access to selected channels and queues.
* Users can set up integrations and use revocable tokens for enhanced security by managing token lifetimes and revocation abilities.

See: "App API keys.":/account/dashboard/api

h2. App integrations

Connect Ably with external services, enabling data to flow between Ably channels and third-party platforms:

* Ably events can trigger webhook requests to external services, or external services can send data to Ably using incoming webhooks.
* Ably enables continuous streaming of events to external services like Amazon Kinesis or Kafka, or can receive inbound streams from external services.
* Ably supports various integration methods, including HTTP callbacks, cloud functions , and streaming services.

See: "Integrations.":/general/integrations

h2. App queues

View, monitor, and adjust Ably queues:

* Queues provide access to manage existing queues, monitor data flow, and adjust settings such as TTL, max length, and region.
* Users can contact Ably support for assistance with increasing limits or customizing their queue settings.

See: "App queues.":/account/dashboard/queues

h2. Push notifications

Manage push notifications in Ably. Create, test, and send notifications to specific devices or channels, and enure everything works correctly.

* Before sending push notifications, you must configure credentials for third-party services like FCM (Android), APNS (iOS), and Web Push (browsers) through the Ably dashboard.
* The push inspector tool helps test and debug push notifications by manually sending notifications and reviewing device, client, and channel registrations.
* Define the title, body, and optional data for your push notifications and choose to target notifications to specific channels, devices, or clients.

See: "Push notifications.":/account/dashboard/notifications

h2. Developer console

Monitor your application's health in realtime by tracking events like connection status changes and channel activity. Connect to channels, publish messages, manage presence, and view event logs for troubleshooting:

* The dev console provides realtime insights into application-wide events like connection status changes, channel activity, and event logs.
* You can connect to channels, publish messages, and manage presence, tracking interactions within the channel through the console.
* The event log table displays detailed records of connection events, which are essential for troubleshooting and diagnosing application issues.
* You can interact with Ably channels using command-line tools like cURL to publish messages and control channel operations.

See: "Developer console.":/account/dashboard/console

h2. App settings

Manage key aspects of your application, including security settings, enabling or disabling the app, configuring rules for channels, and setting up protocol support for different SDKs:

* Manage settings like App ID, name, security (TLS), and enable/disable the application, affecting its ability to accept connections and use Ably services.
* Set rules for specific channels or namespaces, such as message persistence, TLS-only access, push notifications, and server-side batching.
* Enable compatibility with various protocols like Pusher, PubNub, and MQTT for communication between client libraries and Ably.
* Manage notifications for channel activity and occupancy to gain realtime insights into channel usage and performance.

See: "App settings.":/account/dashboard/settings
