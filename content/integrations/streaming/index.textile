Ably's Firehose streams your realtime data, published within the Ably platform, directly to another streaming or queueing service. For example, any device that publishes messages on a channel can immediately stream those messages to "Amazon Kinesis":/docs/integrations/streaming/kinesis, allowing you to process the data in realtime. Using configurable rules, you can stream various "data-sources":#sources, including messages, presence events, occupancy, and channel lifecycle events.

Unlike "channels":/docs/channels, which follow a "pub/sub":/docs/platform#pub-sub pattern, a publisher delivers each message to any number of subscribers. Firehose delivers each message once to your streaming or queueing server.

Each message is delivered once to your streaming or queueing server, making the system well-suited for asynchronous processing of realtime data published by Ably. For example, workers consuming data from your stream or queue can persist each message of a live "chat":/docs/chat to your own database, start publishing updates when a channel becomes active, or trigger an event if a device submits a location indicating it has reached its destination.

There are two ways to create a Firehose through Ably:

* Using the "Ably dashboard":#dash.
* Using the "Control API":#api.

h2(#dash). Create a Firehose rules using the Ably dashboard

You can create Firehose rules in your "dashboard:":https://ably.com/dashboard/any the following services:

* "Amazon Kinesis":/docs/integrations/streaming/kinesis
* "Amazon SQS":/docs/integrations/streaming/sqs
* "AMQP":/docs/integrations/streaming/amqp
* "Apache Kafka":/docs/integrations/streaming/kafka
* "Apache Pulsar":/docs/integrations/streaming/pulsar

h3(#sources). Firehose event sources

You can configure webhooks to listen for the following event types:

- @channel.lifecycle@ := Triggered when a channel is created or discarded. Note, the channel is created in the Ably service when the client "subscribes":/docs/getting-started/quickstart?#step-3 to it.
- @channel.message@ := Triggered when "messages":/docs/messages are published.
- @channel.occupancy@ := Triggered when the number of users in a channel "changes":/docs/channels/options#subscribe-occupancy.
- @channel.presence@ := Triggered when users enter, leave, or update their "presence":/docs/presence-occupancy/occupancy.

h2(#api). Create rule using the Control API

To create a new "rule":/docs/account/control-api#examples-rules for an Ably application, send a POST request to @/apps/{app_id}/rules@ with the rule configuration in the request body.

The following example creates a rule that sends messages from channels matching the regular expression @^my-channel.*@ to a webhook URL:

```[text]
POST /apps/{app_id}/rules
Authorization: Bearer <your_token>
Content-Type: application/json

{
  "status": "enabled",
  "ruleType": "http",
  "requestMode": "single",
  "source": {
    "channelFilter": "^my-channel.*",
    "type": "channel.message"
  },
  "target": {
    "url": "https://example.com/webhooks",
    "headers": [],
    "signingKeyId": "bw66AB",
    "enveloped": true,
    "format": "json"
  }
}
```
