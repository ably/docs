Build a streaming AI chat application using the Anthropic SDK directly with Ably AI Transport. This approach uses the core transport - you work with the Anthropic Messages API and stream tokens through an Ably channel.

<Aside data-type='note'>
This quickstart uses the core transport directly. If you're using Vercel AI SDK with Anthropic models, see the [Vercel AI SDK quickstart](/docs/ai-transport/getting-started/vercel-ai-sdk) for a simpler integration path.
</Aside>

## Prerequisites <a id="prerequisites"/>

- Node.js 20+
- An [Ably account](https://ably.com/sign-up) with an API key
- An Anthropic API key

## Install dependencies <a id="install"/>

<Code>
```shell
npm install ably @anthropic-ai/sdk express
```
</Code>

## Set up environment variables <a id="env"/>

<Code>
```shell
ABLY_API_KEY=your-ably-api-key
ANTHROPIC_API_KEY=your-anthropic-api-key
```
</Code>

## Step 1: Create the server <a id="server"/>

The server receives user messages, creates a turn, invokes the Anthropic Messages API with streaming, and publishes tokens through the Ably channel.

Create a file called `server.mjs`:

<Code>
```javascript
import Ably from 'ably'
import Anthropic from '@anthropic-ai/sdk'
import express from 'express'

const ably = new Ably.Realtime({ key: process.env.ABLY_API_KEY })
const anthropic = new Anthropic()

const app = express()
app.use(express.json())

app.post('/api/chat', async (req, res) => {
  const { channelName, turnId, clientId, messages } = req.body
  const channel = ably.channels.get(channelName)

  // Signal turn start
  await channel.publish('turn-start', JSON.stringify({ turnId, clientId }))

  // Publish user message
  const userMessage = messages[messages.length - 1]
  await channel.publish('user-message', JSON.stringify(userMessage))

  res.json({ status: 'ok' })

  // Stream response after HTTP response
  const stream = anthropic.messages.stream({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 1024,
    messages: messages.map((m) => ({ role: m.role, content: m.content })),
  })

  for await (const event of stream) {
    if (event.type === 'content_block_delta' && event.delta.type === 'text_delta') {
      await channel.publish('token', event.delta.text)
    }
  }

  await channel.publish('turn-end', JSON.stringify({ turnId, reason: 'complete' }))
})

app.listen(3000, () => console.log('Server running on port 3000'))
```
</Code>

## Step 2: Create the client <a id="client"/>

Create a file called `client.mjs`:

<Code>
```javascript
import Ably from 'ably'

const ably = new Ably.Realtime({ authUrl: '/api/auth/ably-token' })
const channel = ably.channels.get('my-chat')

channel.subscribe((message) => {
  if (message.name === 'token') {
    process.stdout.write(message.data)
  } else if (message.name === 'turn-end') {
    console.log('\n[Turn complete]')
  }
})

const response = await fetch('http://localhost:3000/api/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    channelName: 'my-chat',
    turnId: crypto.randomUUID(),
    clientId: 'user-1',
    messages: [{ role: 'user', content: 'What are durable sessions?' }],
  }),
})
```
</Code>

<Aside data-type='note'>
This example uses `authUrl` for token authentication. Never expose Ably API keys in client code. See [Authentication](/docs/ai-transport/how-it-works/authentication) for details.
</Aside>

<Aside data-type='note'>
This example shows the basic integration pattern using raw Ably channel operations. For production use with conversation history, branching, and multi-client sync, build on the [core transport](/docs/ai-transport/how-it-works/transport) with a custom codec or use the [Vercel AI SDK integration](/docs/ai-transport/getting-started/vercel-ai-sdk).
</Aside>
