<Code>
```shell
ably channels publish <channel> <message> [flags]
```
</Code>

Use the `ably channels publish` command to publish a message to a channel.

Use another terminal instance to subscribe to the channel using the [`ably channels subscribe`](/docs/cli/channels/subscribe) command in order to receive published messages.

You can also use the [`ably channels batch-publish`](/docs/cli/channels/batch-publish) command to publish to multiple channels in a single request.

## Arguments

Pass the following arguments to the `ably channels publish` command:

### `<channel>` <a id="channel"/>

The channel name to publish to, for example: `example-channel`.

### `<message>` <a id="message"/>

The message to publish in plain text or JSON format, for example:

* `"Hello!"`
* `{"name":"my-event","data":"Hello!"}`

You can also use interpolation in the message body:

* `{{.Count}}` will display the message number, for example: `Message {{.Count}} of 5`.
* `{{.Timestamp}}` will display the current Unix timestamp in milliseconds, for example: `Message sent at {{.Timestamp}}`.

## Flags

You can optionally pass the following flags to the `ably channels publish` command:

### `--count | -c` <a id="count"/>

The number of messages to publish. The default is `1`.

### `--delay | -d` <a id="delay"/>

The delay between messages being published, in milliseconds. The default is `40`.

### `--encoding | -e` <a id="encoding"/>

The encoding of the message. Common encodings include `json`, `utf-8`, and `base64`.

### `--name | -n` <a id="name"/>

The event name for the message, if not specified in the message JSON.

### `--transport` <a id="transport"/>

The transport method to use to publish the message. Options are `rest` or `realtime`. The default is `rest`.

## Global flags

The following global flags are supported using the `ably channels publish` command:

* [`--token`](/docs/cli#token)
* [`--json`](/docs/cli#json)
* [`--pretty-json`](/docs/cli#pretty-json)
* [`--api-key`](/docs/cli#api-key)
* [`--access-token`](/docs/cli#access-token)
* [`--client-id`](/docs/cli#client-id)
* [`--env`](/docs/cli#env)
* [`--host`](/docs/cli#host)
* [`--endpoint`](/docs/cli#endpoint)
* [`--verbose`](/docs/cli#verbose)

## Examples

### Publish a plain text message

Publish a simple text message:

<Code>
```shell
ably channels publish example-channel "Hello!"
```
</Code>

**Output:**
<Code>
```text
Using: App=example-app (Ab1CdE) • Key=example-key (Ab1CdE.FgHiJk)

✓ Message published successfully to channel "example-channel".
```
</Code>

### Publish a JSON message

Publish a JSON message with an event name specified:

<Code>
```shell
ably channels publish example-channel '{"name":"my-event","data":"Hello!"}'
```
</Code>

Specify the event name separately using the `--name` flag:

<Code>
```shell
ably channels publish --name my-event example-channel '{"data":"Hello!"}'
```
</Code>

### Publish multiple messages

Publish 3 messages using the `--count` flag:

<Code>
```shell
ably channels publish --count 3 example-channel "Message {{.Count}} at {{.Timestamp}}"
```
</Code>

**Output:**
<Code>
```text
Using: App=example-app (Ab1CdE) • Key=example-key (Ab1CdE.FgHiJk)

Publishing 3 messages with 40ms delay...
✓ Message 1 published successfully to channel "example-channel".
Progress: 1/3 messages published (0 errors)
✓ Message 2 published successfully to channel "example-channel".
Progress: 2/3 messages published (0 errors)
✓ Message 3 published successfully to channel "example-channel".
✓ 3/3 messages published successfully.
```
</Code>

### Publish with custom delay

Publish messages with a 1 second delay between them:

<Code>
```shell
ably channels publish --count 5 --delay 1000 example-channel "Delayed message {{.Count}}"
```
</Code>
