Build a streaming AI chat application using the OpenAI SDK with Ably Pub/Sub. This quickstart shows the simplest possible approach: call the OpenAI Responses API on your server and stream each token to connected clients through an Ably channel.

<Aside data-type='note'>
This quickstart uses Ably Pub/Sub directly to stream tokens. For a full-featured integration with structured turns, codec-based message encoding, and conversation history, see the [Vercel AI SDK quickstart](/docs/ai-transport/getting-started/vercel-ai-sdk).
</Aside>

## Prerequisites <a id="prerequisites"/>

To follow this guide you will need:

- [Node.js](https://nodejs.org/) 20 or later.
- An [Ably account](https://ably.com/sign-up) with an API key.
- An [OpenAI API key](https://platform.openai.com/api-keys).

## Install dependencies <a id="install"/>

<Code>
```shell
npm install ably openai express
```
</Code>

## Step 1: Create the server <a id="server"/>

Create a file called `server.mjs`.

The server receives user messages via HTTP, calls the OpenAI Responses API, and publishes each streamed token to an Ably channel. Any number of clients subscribed to that channel will receive the tokens in real time.

<Code>
```javascript
import Ably from 'ably';
import OpenAI from 'openai';
import express from 'express';

const ably = new Ably.Realtime({ key: process.env.ABLY_API_KEY });
const openai = new OpenAI();

const app = express();
app.use(express.json());

app.post('/api/chat', async (req, res) => {
  const { channelName, messages } = req.body;
  const channel = ably.channels.get(channelName);

  const stream = await openai.responses.create({
    model: 'gpt-4o',
    input: messages.map((m) => ({ role: m.role, content: m.content })),
    stream: true,
  });

  // Respond immediately so the client knows the request was accepted
  res.json({ status: 'streaming' });

  // Stream each token through the Ably channel
  for await (const event of stream) {
    if (event.type === 'response.content_part.delta') {
      await channel.publish('token', event.delta);
    }
  }

  // Signal that the response is complete
  await channel.publish('status', 'complete');
});

app.listen(3000, () => console.log('Server running on port 3000'));
```
</Code>

<Aside data-type='note'>
This example streams raw text tokens through Ably Pub/Sub. For production use cases that need structured message types, turn management, and conversation history, use the full AI Transport SDK with a framework adapter. The [Vercel AI SDK quickstart](/docs/ai-transport/getting-started/vercel-ai-sdk) is the fastest path to that.
</Aside>

## Step 2: Create the client <a id="client"/>

Create a file called `client.mjs`.

The client subscribes to the Ably channel to receive streamed tokens. This example uses a Node.js client for simplicity. In a browser app, you would subscribe to the channel in the same way and append tokens to the DOM.

<Code>
```javascript
import Ably from 'ably';

const ably = new Ably.Realtime({ authUrl: '/api/auth/ably-token' });
const channel = ably.channels.get('my-chat');

// Subscribe to streamed tokens
channel.subscribe('token', (message) => {
  process.stdout.write(message.data);
});

channel.subscribe('status', (message) => {
  if (message.data === 'complete') {
    console.log('\n[Response complete]');
  }
});

// Send a user message
const response = await fetch('http://localhost:3000/api/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    channelName: 'my-chat',
    messages: [{ role: 'user', content: 'Hello, what is AI Transport?' }],
  }),
});
```
</Code>

<Aside data-type='note'>
This example uses `authUrl` for token authentication. Never expose Ably API keys in client code. See [Authentication](/docs/ai-transport/how-it-works/authentication) for details.
</Aside>
