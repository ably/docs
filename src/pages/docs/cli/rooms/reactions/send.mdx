<Code>
```shell
ably rooms reactions send <room> <emoji> [flags]
```
</Code>

Use the `ably rooms reactions send` command to send a reaction in a chat room. Room reactions are not tied to a specific message, and are for users to show their sentiment as to what is happening within the room.

Use another terminal instance and the [`ably rooms reactions subscribe`](/docs/cli/rooms/reactions/subscribe) command to view reactions as they are sent.

## Arguments

### `<room>` <a id="room"/>

The room to send the reaction to, for example: `example-room`.

### `<emoji>` <a id="emoji"/>

The emoji reaction to send, for example: `üëç`, `‚ù§Ô∏è`, or `üòÇ`.

## Flags

You can optionally pass the following flags to the `ably rooms reactions send` command:

### `--metadata` <a id="metadata"/>

Additional metadata to send with the reaction as a JSON string, for example: `{"source":"cli","timestamp":123456789}`.

## Global flags

The following global flags are supported using the `ably rooms reactions send` command:

* [`--token`](/docs/cli#token)
* [`--json`](/docs/cli#json)
* [`--pretty-json`](/docs/cli#pretty-json)
* [`--api-key`](/docs/cli#api-key)
* [`--access-token`](/docs/cli#access-token)
* [`--client-id`](/docs/cli#client-id)
* [`--env`](/docs/cli#env)
* [`--host`](/docs/cli#host)
* [`--endpoint`](/docs/cli#endpoint)
* [`--verbose`](/docs/cli#verbose)
