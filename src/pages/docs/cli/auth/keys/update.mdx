Use the `ably auth keys update` command to update an API key's properties.

## Synopsis

<Code>
```shell
ably auth keys update <key-name> [flags]
```
</Code>

## Arguments

### `key-name` <a id="key-name"/> <RequiredBadge />

The key name (`APP_ID.KEY_ID`) of the API key to update.

## Flags

### `--app` <a id="app"/>

The app ID the key belongs to. Uses the currently selected app if not specified.

### `--name` <a id="name"/>

The new name for the API key.

### `--capabilities` <a id="capabilities"/>

New [capabilities](/docs/auth/capabilities) for the key, specified as either a JSON object for per-channel capabilities or a comma-separated list applied to all channels.

## Global flags

* [`--json`](/docs/cli#json) or [`--pretty-json`](/docs/cli#pretty-json) — JSON output. Mutually exclusive flags.
* [`--verbose`](/docs/cli#verbose) — Enable verbose logging. Can be combined with `--json` or `--pretty-json`.

## Examples

Rename an API key:

<Code>
```shell
ably auth keys update "aBcDeF.gHiJkL" --name "Production Key"
```
</Code>

Update the capabilities for a key:

<Code>
```shell
ably auth keys update "aBcDeF.gHiJkL" --capabilities "publish,subscribe"
```
</Code>

Update both the name and capabilities for a key in a single command:

<Code>
```shell
ably auth keys update APP_ID.KEY_ID --name "New Name" --capabilities "publish,subscribe"
```
</Code>

Update the capabilities for a key using a JSON object for per-channel control:

<Code>
```shell
ably auth keys update APP_ID.KEY_ID --name "New Name" --capabilities '{"channel1":["publish"],"channel2":["subscribe"]}'
```
</Code>

Update a key and output the result in JSON format:

<Code>
```shell
ably auth keys update APP_ID.KEY_ID --name "New Name" --json
```
</Code>
