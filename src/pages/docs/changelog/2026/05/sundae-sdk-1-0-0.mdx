We are delighted to announce the **first stable release** of the Sundae SDK. This release graduates the _Flavour API_ out of preview, adds realtime scoop streaming, and introduces presence-aware sprinkles.

This is a dummy entry used to verify that the standard documentation components render correctly inside a changelog page.

<Aside data-type='new'>
The Flavour API is now generally available and follows semantic versioning from this release onward.
</Aside>

## What's new <a id="whats-new"/>

The headline features in this release:

- **Realtime scoop streaming** so every cone stays in sync across devices.
- A typed `Flavour` model with built-in nutrition metadata.
- Presence-aware sprinkles that only render for connected diners.
- A new `meltWarning` event for when ambient temperature climbs.

You can mix and match flavours per channel. For more on channels, see [channels](/docs/channels) and the [presence](/docs/presence/occupancy) docs.

### Scoop a flavour <a id="scoop"/>

Create a client and scoop your first flavour. The highlighted lines show the new options object:

<Code>
```javascript highlight="+3-4"
const sundae = new Sundae.Realtime('{{API_KEY}}');
const cone = sundae.cones.get('{{RANDOM_CHANNEL_NAME}}');
await cone.scoop('pistachio', { sprinkles: true, waffle: 'double' });
await cone.scoop('stracciatella');
```

```python highlight="+3-4"
sundae = Sundae.Realtime('{{API_KEY}}')
cone = sundae.cones.get('{{RANDOM_CHANNEL_NAME}}')
cone.scoop('pistachio', sprinkles=True, waffle='double')
cone.scoop('stracciatella')
```
</Code>

<Aside data-type='note'>
Inline code such as `cone.scoop()` renders with the monospace inline style, while fenced blocks above become language tabs.
</Aside>

### Subscribe to melt warnings <a id="melt-warnings"/>

Listen for the new `meltWarning` event to refrigerate just in time:

<Code>
```javascript
cone.on('meltWarning', (event) => {
  console.log(`Hurry! ${event.flavour} is melting at ${event.tempC}°C`);
});
```
</Code>

<Aside data-type='important'>
Melt warnings fire at most once per minute per cone. Do not rely on them for precise temperature monitoring of industrial freezers.
</Aside>

## Flavour reference <a id="flavour-reference"/>

The `Flavour` object now exposes the following properties:

| Property | Type | Description |
| --- | --- | --- |
| `name` | string | The display name of the flavour, for example `pistachio`. |
| `intensity` | number | A 1–10 scale of how bold the flavour is. |
| `vegan` | boolean | Whether the flavour contains no dairy. |
| `allergens` | string[] | A list of allergens, for example `['nuts', 'soy']`. |

## Configure your parlour <a id="configure"/>

You can configure default sprinkles in the dashboard or programmatically.

<Tabs>
<Tab value="dashboard" label="Dashboard">

In your parlour settings:

1. Open the **Toppings** tab.
2. Toggle **Sprinkles on by default**.
3. Choose a default `waffle` type.
4. Click **Save** to apply the rule.

</Tab>
<Tab value="config-api" label="Config API">

Send a `POST` request to create a default-toppings rule:

<Code>
```shell
curl -X POST https://parlour.example.com/v1/toppings \
  -H "Authorization: Bearer {ACCESS_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{ "sprinkles": true, "waffle": "double" }'
```
</Code>

</Tab>
</Tabs>

The response is a standard JSON payload:

<Code>
```json
{
  "id": "rule_42",
  "sprinkles": true,
  "waffle": "double",
  "created": "2026-05-21T09:00:00Z"
}
```
</Code>
