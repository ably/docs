Use the `ably auth keys revoke` command to permanently revoke an API key.

## Synopsis

<Code>
```shell
ably auth keys revoke <key-name> [flags]
```
</Code>

## Arguments

### `key-name` <a id="key-name"/> `(Required)`

The key name (`APP_ID.KEY_ID`) of the API key to revoke.

## Flags

### `--app` <a id="app"/>

The app ID the key belongs to. Uses the currently selected app if not specified.

### `--force | -f` <a id="force"/>

Skip the confirmation prompt and revoke the key immediately. Required when using `--json`.

## Global flags

* [`--json`](/docs/cli#json) or [`--pretty-json`](/docs/cli#pretty-json) — mutually exclusive output format flags.
* [`--verbose`](/docs/cli#verbose) — can be combined with either output format flag.

## Examples

Revoke an API key:

<Code>
```shell
ably auth keys revoke "aBcDeF.gHiJkL"
```
</Code>

Force revoke without confirmation:

<Code>
```shell
ably auth keys revoke "aBcDeF.gHiJkL" --force
```
</Code>

Revoke a key using separate key ID and app flag:

<Code>
```shell
ably auth keys revoke KEY_ID --app APP_ID
```
</Code>

Revoke a key and output the result in JSON format:

<Code>
```shell
ably auth keys revoke APP_ID.KEY_ID --json
```
</Code>

Revoke a key and output the result in formatted JSON:

<Code>
```shell
ably auth keys revoke APP_ID.KEY_ID --pretty-json
```
</Code>
