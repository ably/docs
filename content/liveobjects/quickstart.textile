This guide shows how to integrate Ably LiveObjects into your application.

You will learn how to:

* Create an Ably account and get an API key.
* Install the Ably SDK.
* Create a channel with LiveObjects functionality enabled.
* Create, update and subscribe to changes on LiveObjects data structures: @LiveMap@ and @LiveCounter@.

h2(#step-0). Create an Ably account

"Sign up":https://ably.com/signup for a free account to get your own API key. Ensure that your API key includes the @subscribe@ and @publish@ "capabilities":/docs/auth/capabilities.

h2(#step-1). Install Ably SDK

blang[javascript].

  LiveObjects is available as part of the Ably Pub/Sub SDK via the dedicated Objects plugin.

blang[javascript].

  The Ably Pub/Sub SDK is available as an "NPM module":https://www.npmjs.com/package/ably:

  ```[sh]
  npm install ably
  ```

  And then import it into your project:

  ```[javascript]
  import * as Ably from 'ably';
  import Objects from 'ably/objects';
  ```

  Alternatively, you can install the Ably Pub/Sub SDK and the Objects plugin via a CDN on an HTML page:

  ```[html]
  <script src="https://cdn.ably.com/lib/ably.min-2.js"></script>
  <script src="https://cdn.ably.com/lib/objects.umd.min-2.js"></script>
  <script>
    // Objects plugin is now available on the global object via the `AblyObjectsPlugin` property
    const Objects = window.AblyObjectsPlugin;
  </script>
  ```

h2(#step-2). Create a client

blang[javascript].

  Create an Ably Realtime client with the Objects plugin:

  ```[javascript]
  const realtimeClient = new Ably.Realtime({ key: '{{API_KEY}}', plugins: { Objects } });
  ```

h2(#step-3). Create a channel

Create a new channel and "attach to it":/docs/channels/states. Attaching to a channel also starts a LiveObjects synchronization sequence where the state on a client is synced with the state on the Ably system.

blang[javascript].

  ```[javascript]
  // OBJECT_* modes are required to enable LiveObjects on a channel.
  // Read more about channel modes: https://ably.com/docs/channels/options#modes
  const channelOptions = { modes: ['OBJECT_SUBSCRIBE', 'OBJECT_PUBLISH'] };
  const channel = realtimeClient.channels.get('my_liveobjects_channel', channelOptions);
  await channel.attach();
  ```

h2(#step-4). Get root object

"channel.objects":/docs/api/realtime-sdk/channels#objects property gives access to the LiveObjects API for a channel. Use it to get the root object - an entry point to your LiveObjects state on a channel. The root object always exists on a channel, and you can get it using the *getRoot* function of LiveObjects.

blang[javascript].

  ```[javascript]
  // The promise resolves once the LiveObjects state is synchronized with the Ably system
  const root = await channel.objects.getRoot();
  ```

h2(#step-5). Create objects

You can create new objects using dedicated functions of the LiveObjects API. To persist them on a channel and share them between clients, you must assign objects to a parent @LiveMap@ instance connected to the root object. The root object itself is a @LiveMap@ instance, so you can assign objects to the root and start building your state tree from there.

blang[javascript].

  ```[javascript]
  const visitsCounter = await channel.objects.createCounter();
  const reactionsMap = await channel.objects.createMap();

  await root.set('visits', visitsCounter);
  await root.set('reactions', reactionsMap);
  ```

h2(#step-6). Subscribe to updates

Subscribe to realtime updates to Live Objects. You will be notified when an object is updated by other clients or by you.

blang[javascript].

  ```[javascript]
  visitsCounter.subscribe(() => {
    console.log('Visits counter updated:', visitsCounter.value());
  });

  reactionsMap.subscribe(() => {
    console.log('Reactions map updated:', [...reactionsMap.entries()]);
  });
  ```

h2(#step-7). Update Live Objects

Update Live Objects and get notified about the changes:

blang[javascript].

  ```[javascript]
  await visitsCounter.increment(5);
  // console: "Visits counter updated: 5"
  await visitsCounter.decrement(2);
  // console: "Visits counter updated: 2"

  await reactionsMap.set('like', 10);
  // console: "Reactions map updated: [['like',10]]"
  await reactionsMap.set('love', 10);
  // console: "Reactions map updated: [['like',10],['love',5]]"
  await reactionsMap.remove('like');
  // console: "Reactions map updated: [['love',5]]"
  ```

<aside data-type='note'>
<p>
  Mutation methods (such as @LiveMap.set@, @LiveCounter.increment@, etc.) do not directly modify the local object state. Instead, they send the intended operation to the Ably system, and the change is applied only when the corresponding realtime operation is echoed back to the client. This means that the state retrieved immediately after a mutation may not reflect the latest updates yet. You will be notified via subscription when the object is updated.
</p>
</aside>
