"Azure Functions":https://azure.microsoft.com/en-gb/services/functions/ provide event-driven serverless compute functions which allow users to easily run code whenever events are sent to it.

To trigger an Azure Function in response to Ably channel events:

* Go to your Ably "dashboard":https://ably.com/dashboard/any and select your app.
* Navigate to the *Integrations* tab.
* Click *New Integration Rule*.
* In the *Select your rule type* section, choose *webhook*.
* Under *Select service integration*, click *Azure Functions*.
* Fill in the required fields:
** *Azure App ID* - The ID of your Azure App.
** *Function Name* - The name of your Azure Function.
* (Optional) Add custom *Headers* as key:value pairs for additional information.
** Click *Another header* to add multiple headers.
* Choose the *Request Mode*:
** "*Single Request*":/docs/integrations/webhooks#single - Sends each event separately.
** "*Batch Request*":/docs/integrations/webhooks#batched - Sends requests in batches (1 second interval, with a max number of messages per envelope).
* In the "*Source*":/docs/integrations/webhooks#sources dropdown, select the type of events that will trigger the rule:
** Message
** Presence
** Channel Lifecycle
** Channel Occupancy
* Add a "*Channel Filter*":/docs/integrations/webhooks#channel-filter using a regular expression to target specific channels. Leave empty to apply to all channels.
* Choose an "*Encoding*":/docs/integrations/webhooks#encoding format:
** JSON
** MsgPack
* Choose if you want messages to be "*Enveloped:*":/docs/integrations/webhooks#enveloped
** Enabled by default. If unchecked, the raw payload will be sent without additional metadata.
* Click *Create* to finish setting up the rule.

<aside data-type='note'>
<p>This integration does not use a raw URL.</p>
</aside>
