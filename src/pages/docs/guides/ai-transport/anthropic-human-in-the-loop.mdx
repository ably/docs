This guide shows you how to build a human-in-the-loop (HITL) approval flow for Anthropic [tool use](https://docs.anthropic.com/en/docs/build-with-claude/tool-use) using Ably as the realtime transport layer. Specifically, when an AI agent calls a tool, the agent publishes an approval request over Ably and waits for a human to approve or reject it before proceeding.

Using Ably to transport approval requests and responses enables you to decouple the AI agent from the human approver, supporting scenarios where approvers are on different devices, networks, or platforms. Ably provides reliable delivery so that approval requests are never lost, even if the approver is temporarily disconnected.

<Aside data-type="further-reading">
To learn more about human-in-the-loop patterns, see the [human-in-the-loop](/docs/ai-transport/messaging/human-in-the-loop) documentation.
</Aside>

## Step 2: Initialize Ably and publish approval requests <a id="step-2"/>

When the agent receives a `tool_use` response from Anthropic, publish the tool call details as an approval request over Ably. The agent then waits for a human to approve or reject the action before proceeding.

Add the Ably client initialization and approval request logic to your agent file:

<Code>
```agent_javascript
import Ably from 'ably';

// Initialize Ably Realtime client
const realtime = new Ably.Realtime({
  key: '{{API_KEY}}',
  echoMessages: false,
});

// Create a channel for approval messages
const channel = realtime.channels.get('ai:{{RANDOM_CHANNEL_NAME}}');

// Track pending approvals
const pendingApprovals = new Map();

// Request human approval for a tool use via Ably
async function requestHumanApproval(toolUse) {
  const approvalPromise = new Promise((resolve, reject) => {
    pendingApprovals.set(toolUse.id, { resolve, reject });
  });

  await channel.publish({
    name: 'approval-request',
    data: {
      tool: toolUse.name,
      arguments: toolUse.input,
    },
    extras: {
      headers: {
        toolCallId: toolUse.id,
      },
    },
  });

  console.log(`Approval request sent for: ${toolUse.name}`);
  return approvalPromise;
}
```

```agent_java
import io.ably.lib.realtime.AblyRealtime;
import io.ably.lib.realtime.Channel;
import io.ably.lib.types.ClientOptions;
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.CompletableFuture;

// Initialize Ably Realtime client
ClientOptions options = new ClientOptions("{{API_KEY}}");
options.echoMessages = false;
AblyRealtime realtime = new AblyRealtime(options);

// Create a channel for approval messages
Channel channel = realtime.channels.get("ai:{{RANDOM_CHANNEL_NAME}}");

// Track pending approvals
Map<String, CompletableFuture<Boolean>> pendingApprovals = new ConcurrentHashMap<>();

// Request human approval for a tool use via Ably
CompletableFuture<Boolean> requestHumanApproval(String toolCallId, String toolName, Map<String, Object> arguments) {
    CompletableFuture<Boolean> future = new CompletableFuture<>();
    pendingApprovals.put(toolCallId, future);

    io.ably.lib.types.Message message = new io.ably.lib.types.Message();
    message.name = "approval-request";
    message.data = Map.of("tool", toolName, "arguments", arguments);
    channel.publish(message);

    System.out.println("Approval request sent for: " + toolName);
    return future;
}
```
</Code>

The agent publishes the tool name and arguments as an `approval-request` message over Ably. The `toolCallId` is included in the message extras as a header, allowing the approval response to be correlated back to the original request.

<Aside data-type="note">
Set [`echoMessages`](/docs/api/realtime-sdk/types#client-options) to `false` on the agent's Ably client to prevent the agent from receiving its own messages, avoiding billing for [echoed messages](/docs/pub-sub/advanced#echo).
</Aside>

## Step 3: Subscribe to approval responses on the agent <a id="step-3"/>

The agent subscribes to `approval-response` messages on the same channel. When a response arrives, it resolves or rejects the pending promise based on the human's decision.

Add the approval response handler to your agent file:

<Code>
```agent_javascript
// Subscribe to approval responses and resolve pending promises
channel.subscribe('approval-response', (message) => {
  const { decision } = message.data;
  const toolCallId = message.extras?.headers?.toolCallId;
  const pending = pendingApprovals.get(toolCallId);

  if (!pending) return;

  if (decision === 'approved') {
    pending.resolve({ approved: true });
  } else {
    pending.reject(new Error('Action rejected by approver'));
  }
  pendingApprovals.delete(toolCallId);
});

// Process a tool use: request approval then execute
async function processToolUse(toolUse) {
  if (toolUse.name === 'publish_blog_post') {
    await requestHumanApproval(toolUse);
    return publishBlogPost(toolUse.input);
  }
  throw new Error(`Unknown tool: ${toolUse.name}`);
}

// Run the agent: send prompt to Anthropic, handle tool use with approval
async function runAgent(prompt) {
  const response = await callAnthropic(prompt);

  const toolUseBlocks = response.content.filter(
    (block) => block.type === 'tool_use',
  );

  const results = [];
  for (const toolUse of toolUseBlocks) {
    try {
      const result = await processToolUse(toolUse);
      results.push({ success: true, result });
    } catch (err) {
      results.push({ success: false, error: err.message });
    }
  }

  return results;
}

// Usage example
const results = await runAgent('Publish the blog post called "Introducing our new API"');
console.log('Results:', results);
realtime.close();
```

```agent_java

channel.subscribe("approval-response", message -> {
    Map<String, Object> data = (Map<String, Object>) message.data;
    String decision = (String) data.get("decision");
    String toolCallId = message.extras != null ?
        (String) message.extras.get("headers", Map.class).get("toolCallId") : null;

    CompletableFuture<Boolean> future = pendingApprovals.get(toolCallId);
    if (future == null) return;

    if ("approved".equals(decision)) {
        future.complete(true);
    } else {
        future.completeExceptionally(new Exception("Action rejected by approver"));
    }
    pendingApprovals.remove(toolCallId);
});

// Run the agent: send prompt to Anthropic, handle tool use with approval
public static void main(String[] args) throws Exception {
    Message response = callAnthropic("Publish the blog post called 'Introducing our new API'");

    for (ContentBlock block : response.content()) {
        if (block.isToolUse()) {
            ToolUseBlock toolUse = block.asToolUse();
            CompletableFuture<Boolean> approval = requestHumanApproval(
                toolUse.id(), toolUse.name(), (Map<String, Object>) toolUse.input()
            );

            try {
                approval.get(); // Wait for approval
                Map<String, Object> result = publishBlogPost(
                    (String) ((Map<String, Object>) toolUse.input()).get("title")
                );
                System.out.println("Result: " + result);
            } catch (Exception e) {
                System.out.println("Rejected: " + e.getMessage());
            }
        }
    }

    realtime.close();
}
```
</Code>

When `requestHumanApproval` is called, the agent blocks until the corresponding approval response arrives over Ably. If the response is `approved`, the tool executes. If `rejected`, the agent skips execution and records the rejection.

## Step 5: Add role-based access control <a id="step-5"/>

In production, not every user should be able to approve every action. Add role-based access control to the agent so it validates the approver's role before accepting the decision.

Update the approval response handler in the agent to check roles:

<Code>
```agent_javascript
// Define role hierarchy and approval policies
const roleHierarchy = ['editor', 'publisher', 'admin'];
const approvalPolicies = {
  publish_blog_post: { minRole: 'publisher' },
};

function canApprove(approverRole, requiredRole) {
  return roleHierarchy.indexOf(approverRole) >= roleHierarchy.indexOf(requiredRole);
}

// Updated subscription with role checking
channel.subscribe('approval-response', (message) => {
  const { decision, approverRole } = message.data;
  const toolCallId = message.extras?.headers?.toolCallId;
  const pending = pendingApprovals.get(toolCallId);

  if (!pending) return;

  // Check if the approver has sufficient permissions
  const policy = approvalPolicies['publish_blog_post'];
  if (approverRole && !canApprove(approverRole, policy.minRole)) {
    pending.reject(
      new Error(`Approver role '${approverRole}' insufficient for '${policy.minRole}'`),
    );
    pendingApprovals.delete(toolCallId);
    return;
  }

  if (decision === 'approved') {
    pending.resolve({ approved: true, approverRole });
  } else {
    pending.reject(new Error(`Action rejected by ${approverRole}`));
  }
  pendingApprovals.delete(toolCallId);
});
```

```agent_python
        return

    # Check if the approver has sufficient permissions
    policy = approval_policies.get('publish_blog_post', {})
    if approver_role and not can_approve(approver_role, policy.get('min_role', '')):
        future.set_exception(
            Exception(f'Approver role "{approver_role}" insufficient for "{policy["min_role"]}"')
        )
        del pending_approvals[tool_call_id]
        return

    if decision == 'approved':
        future.set_result({'approved': True, 'approver_role': approver_role})
    else:
        future.set_exception(Exception(f'Action rejected by {approver_role}'))
    del pending_approvals[tool_call_id]


await channel.subscribe('approval-response', on_approval_response)
```

```agent_java
// Define role hierarchy and approval policies
List<String> roleHierarchy = List.of("editor", "publisher", "admin");
Map<String, String> approvalPolicies = Map.of("publish_blog_post", "publisher");

boolean canApprove(String approverRole, String requiredRole) {
    return roleHierarchy.indexOf(approverRole) >= roleHierarchy.indexOf(requiredRole);
}

// Updated subscription with role checking

    Map<String, Object> data = (Map<String, Object>) message.data;
    String decision = (String) data.get("decision");
    String approverRole = (String) data.get("approverRole");
    String toolCallId = message.extras != null ?
        (String) ((Map<String, Object>) message.extras.get("headers", Map.class)).get("toolCallId") : null;

    CompletableFuture<Boolean> future = pendingApprovals.get(toolCallId);
    if (future == null) return;

    // Check if the approver has sufficient permissions
    String requiredRole = approvalPolicies.get("publish_blog_post");
    if (approverRole != null && !canApprove(approverRole, requiredRole)) {
        future.completeExceptionally(
            new Exception("Approver role '" + approverRole + "' insufficient for '" + requiredRole + "'")
        );
        pendingApprovals.remove(toolCallId);
        return;
    }

    if ("approved".equals(decision)) {
        future.complete(true);
    } else {
        future.completeExceptionally(new Exception("Action rejected by " + approverRole));
    }
    pendingApprovals.remove(toolCallId);
});
```

The role hierarchy defines an ordered list of roles with increasing permissions. When an approval response arrives, the agent checks whether the approver's role meets the minimum requirement defined in `approvalPolicies`. If the role is insufficient, the approval is rejected regardless of the decision.
