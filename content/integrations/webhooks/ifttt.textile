"IFTTT":https://ifttt.com/maker_webhooks enables conditional chains, helping to combine various services together. Ably can integrate into IFTTT through their webhooks feature.

To trigger an IFTTT applet using Ably channel events:

* Go to your Ably "dashboard":https://ably.com/dashboard/any and select your app.
* Navigate to the *Integrations* tab.
* Click *New Integration Rule*.
* In the *Select your rule type* section, choose *webhook*.
* Under *Select service integration*, click *IFTTT*.
* Click *Open IFTTT webhook Settings* to connect webhooks with your IFTTT account.
* In your IFTTT account, go to the *webhook Service Documentation* and copy the *webhook key*.
* Paste the key into the *IFTTT webhook key* field.
* Enter the *Event name* — this must match the name used when creating your IFTTT applet.
* In the "*Source*":/docs/integrations/webhooks#sources dropdown, select which type of Ably event should trigger the rule:
** Message
** Presence
** Channel Lifecycle
** Channel Occupancy
* (Optional) Add a "*Channel Filter*":/docs/integrations/webhooks#channel-filter using a regular expression to apply the rule to specific channels. Leave empty to apply to all.
* Choose an "*Encoding*":/docs/integrations/webhooks#encoding format:
** JSON
** MsgPack
* Click *Create* to finish setting up the rule.

h2(#restrictions). IFTTT webhook restrictions

IFTTT has limitations on the data it can process. All payloads must be @JSON@ and use only the keys @value1@, @value2@, or @value3@. Any other format or additional keys will not be processed.

As a result, we do not support "enveloping":/docs/integrations/webhooks#enveloped, “batching”:/docs/integrations/webhooks#batching, or MsgPack encoding for IFTTT. Additionally, protocols requiring decoding, for example "MQTT":/docs/protocols/mqtt, are not supported with IFTTT.

To ensure data is processed by IFTTT, it must match the required IFTTT structure. The following example shows the headers and payload sent to IFTTT when a message is sent to a channel:

```[json]
{
  "value1" :"data I want to send 1",
  "value2" :"data I want to send 2",
  "value3" :"data I want to send 3"
}
```

For a "message data":/docs/api/realtime-sdk/messages#data or "presence message data":/docs/api/realtime-sdk/presence#presence-message of @{ "value1": "My first message", "value2": "My second message" }@, the following would be sent to your IFTTT endpoint.

The following example shows the headers and payload sent to IFTTT when a message is sent to a channel:

```[text]
host: https://maker.ifttt.com/trigger/{YOUR_EVENT}/with/key/{YOUR_IFTTT_KEY}
content-type: application/json
x-ably-envelope-appid: {YOUR_APP_ID}
x-ably-envelope-channel: {YOUR_CHANNEL}
x-ably-envelope-rule-id: {YOUR_RULE_ID}
x-ably-envelope-site: {ably-server-location}
x-ably-envelope-source: channel.message
x-ably-message-encoding: json
x-ably-message-id: {UNIQUE_ABLY_MESSAGE_ID}
x-ably-message-timestamp: {TIMESTAMP_ORIGINAL_MESSAGE_WAS_SENT}
x-ably-version: 1.2
content-length: 18
connection: keep-alive
```

The following is the payload sent to IFTTT:

```[json]
{
  "value1": "My first message",
  "value2": "My second message"
}
```
