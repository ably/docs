The `PushChannel` object lets a device subscribe to and unsubscribe from push notifications published on a channel, and list the push subscriptions on that channel. Access it via `channel.push`.

<Code>
```javascript
const pushChannel = channel.push;
```
</Code>

## Subscribe this device <a id="subscribe-device"/>

<MethodSignature>{`pushChannel.subscribeDevice(): Promise<void>`}</MethodSignature>

Subscribe the local device to the channel's push notifications.

<Code>
```javascript
await channel.push.subscribeDevice();
```
</Code>

### Returns <a id="subscribe-device-returns"/>

`Promise<void>`

Returns a promise. The promise is fulfilled when the subscription has been created, or rejected with an [`ErrorInfo`](/docs/pub-sub/api/javascript/rest/rest-client#errorinfo) object.

## Subscribe all devices with this clientId <a id="subscribe-client"/>

<MethodSignature>{`pushChannel.subscribeClient(): Promise<void>`}</MethodSignature>

[Subscribe all devices associated with this client's `clientId`](/docs/push/publish#sub-channels) to the channel's push notifications.

<Code>
```javascript
await channel.push.subscribeClient();
```
</Code>

### Returns <a id="subscribe-client-returns"/>

`Promise<void>`

Returns a promise. The promise is fulfilled when the subscription has been created, or rejected with an [`ErrorInfo`](/docs/pub-sub/api/javascript/rest/rest-client#errorinfo) object.

## List subscriptions on this channel <a id="list-subscriptions"/>

<MethodSignature>{`pushChannel.listSubscriptions(params?: Record<string, string>): Promise<PaginatedResult<PushChannelSubscription>>`}</MethodSignature>

Lists push subscriptions on the channel by querying the REST API's [list channel subscriptions endpoint](/docs/api/rest-api#list-channel-subscriptions). These subscriptions can be a list of client (`clientId`) subscriptions, device (`deviceId`) subscriptions, or a combination of both. This method requires clients to have the [Push Admin capability](/docs/push#push-admin).

<Code>
```javascript
const subscriptions = await channel.push.listSubscriptions({ deviceId: '<device-id>' });
```
</Code>

### Parameters <a id="list-subscriptions-params"/>

The `listSubscriptions()` method takes the following parameters:

<Table id='ListSubscriptionsParameters'>

| Parameter | Required | Description | Type |
| --- | --- | --- | --- |
| params | Optional | An object containing key-value pairs to filter subscriptions by. Can contain `clientId`, `deviceId`, or a combination of both, and a `limit` on the number of subscriptions returned (up to 1,000). | `Record<String, String>` |

</Table>

### Returns <a id="list-subscriptions-returns"/>

`Promise<PaginatedResult<PushChannelSubscription>>`

Returns a promise. The promise is fulfilled with a [`PaginatedResult`](/docs/pub-sub/api/javascript/rest/rest-client#PaginatedResult) containing an array of `PushChannelSubscription` objects, which supports pagination using its `next` and `first` methods, or rejected with an [`ErrorInfo`](/docs/pub-sub/api/javascript/rest/rest-client#errorinfo) object.

<Table id='PushChannelSubscription'>

| Property | Description | Type |
| --- | --- | --- |
| channel | The channel that this push notification subscription is associated with. | String |
| deviceId | The device with this identifier is linked to this channel subscription. When present, `clientId` is never present. | String or Undefined |
| clientId | Devices with this [client identifier](/docs/auth/identified-clients) are included in this channel subscription. When present, `deviceId` is never present. | String or Undefined |

</Table>
