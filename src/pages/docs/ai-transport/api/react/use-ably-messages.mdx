Subscribe to the raw `Ably.InboundMessage` stream from the transport's tree. Messages are appended in arrival order. Use when you need to inspect channel-level metadata (serials, raw headers) that the codec layer would otherwise hide. Most applications should use [`useView`](/docs/ai-transport/api/react/use-view) instead, which returns decoded domain messages.

This hook must be used within a [`TransportProvider`](/docs/ai-transport/api/react/providers#TransportProvider) or a [`ChatTransportProvider`](/docs/ai-transport/api/react/providers#ChatTransportProvider).

<Code>
```react
import { useAblyMessages } from '@ably/ai-transport/react';

function RawLog() {
  const messages = useAblyMessages();
  return <pre>{messages.map((m) => `${m.serial} ${m.name}`).join('\n')}</pre>;
}
```
</Code>

## Parameters

<Table id='UseAblyMessagesProps'>

| Prop | Required | Description | Type |
| --- | --- | --- | --- |
| transport | Optional | Transport to subscribe to. Defaults to the nearest provider when omitted. | `ClientTransport<TEvent, TMessage>` |
| skip | Optional | When `true`, skip all subscriptions and return an empty array. | Boolean |

</Table>

## Returns

`Ably.InboundMessage[]`

The accumulated raw Ably messages in chronological order. The array grows as messages arrive.

## Example

<Code>
```react
import { useAblyMessages } from '@ably/ai-transport/react';

function ChannelTrace() {
  const raw = useAblyMessages();
  return (
    <ul>
      {raw.map((m) => (
        <li key={m.serial}>
          <code>{m.serial}</code> {m.name} ({m.action})
        </li>
      ))}
    </ul>
  );
}
```
</Code>
