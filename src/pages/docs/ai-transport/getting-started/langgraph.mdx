Build a streaming AI agent using LangGraph with Ably AI Transport. LangGraph handles agent orchestration and state management. AI Transport adds a durable session layer so your agents stream reliably to any number of connected clients.

<Aside data-type='note'>
This quickstart uses the core transport directly. If you're using Vercel AI SDK alongside LangGraph, see the [Vercel AI SDK quickstart](/docs/ai-transport/getting-started/vercel-ai-sdk).
</Aside>

## Prerequisites <a id="prerequisites"/>

- Node.js 20+
- An [Ably account](https://ably.com/sign-up) with an API key
- An Anthropic API key (LangGraph uses it as the LLM provider in this example)

## Install dependencies <a id="install"/>

<Code>
```shell
npm install ably @langchain/langgraph @langchain/anthropic @langchain/core express
```
</Code>

## Set up environment variables <a id="env"/>

<Code>
```shell
ABLY_API_KEY=your-ably-api-key
ANTHROPIC_API_KEY=your-anthropic-api-key
```
</Code>

## Step 1: Create the server <a id="server"/>

File: `server.mjs`

The server runs a LangGraph agent and streams its output through an Ably channel. LangGraph manages the agent's state machine - AI Transport handles delivery to clients.

<Code>
```javascript
import Ably from 'ably'
import { ChatAnthropic } from '@langchain/anthropic'
import { StateGraph, MessagesAnnotation } from '@langchain/langgraph'
import express from 'express'

const ably = new Ably.Realtime({ key: process.env.ABLY_API_KEY })

const model = new ChatAnthropic({ model: 'claude-sonnet-4-20250514' })

const graph = new StateGraph(MessagesAnnotation)
  .addNode('agent', async (state) => {
    const response = await model.invoke(state.messages)
    return { messages: [response] }
  })
  .addEdge('__start__', 'agent')
  .compile()

const app = express()
app.use(express.json())

app.post('/api/chat', async (req, res) => {
  const { channelName, turnId, clientId, messages } = req.body
  const channel = ably.channels.get(channelName)

  await channel.publish('turn-start', JSON.stringify({ turnId, clientId }))
  await channel.publish('user-message', JSON.stringify(messages[messages.length - 1]))

  res.json({ status: 'ok' })

  const stream = await graph.stream(
    { messages: messages.map((m) => ({ role: m.role, content: m.content })) },
    { streamMode: 'messages' },
  )

  for await (const [message, metadata] of stream) {
    if (message.content) {
      await channel.publish('token', typeof message.content === 'string' ? message.content : JSON.stringify(message.content))
    }
  }

  await channel.publish('turn-end', JSON.stringify({ turnId, reason: 'complete' }))
})

app.listen(3000, () => console.log('Server running on port 3000'))
```
</Code>

## Step 2: Create the client <a id="client"/>

File: `client.mjs`

<Code>
```javascript
import Ably from 'ably'

const ably = new Ably.Realtime({ authUrl: '/api/auth/ably-token' })
const channel = ably.channels.get('my-agent-session')

channel.subscribe((message) => {
  if (message.name === 'token') {
    process.stdout.write(message.data)
  } else if (message.name === 'turn-end') {
    console.log('\n[Agent finished]')
  }
})

await fetch('http://localhost:3000/api/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    channelName: 'my-agent-session',
    turnId: crypto.randomUUID(),
    clientId: 'user-1',
    messages: [{ role: 'user', content: 'What is LangGraph?' }],
  }),
})
```
</Code>

<Aside data-type='note'>
This example uses `authUrl` for token authentication. Never expose Ably API keys in client code. See [Authentication](/docs/ai-transport/how-it-works/authentication) for details.
</Aside>

<Aside data-type='note'>
LangGraph's architecture - where the agent manages its own state machine - pairs naturally with AI Transport's durable sessions. The agent doesn't need to worry about connection management, multi-client delivery, or stream recovery. It publishes events to the channel and AI Transport handles the rest.
</Aside>
