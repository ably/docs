Ably AI Transport adds a durable session layer to your LangGraph application. LangGraph handles agent orchestration and state management - AI Transport handles delivery to clients with resilient streaming and multi-device sync.

Ready to build? [Get started with LangGraph](/docs/ai-transport/getting-started/langgraph) in 5 minutes.

## Understand what each layer does <a id="layers"/>

LangGraph and AI Transport solve different problems. They stack cleanly.

| LangGraph | Ably AI Transport |
| --- | --- |
| Graph-based agent orchestration | Durable sessions (Ably channels) |
| State machines and checkpoints | Multi-device fan-out |
| Multi-agent coordination | Reconnection and recovery |
| Tool calling and routing | Ordering and persistence |
| Streaming (`streamMode: 'messages'`) | Bidirectional control |
| | Active turn tracking |

LangGraph manages the agent's execution graph. AI Transport manages how that execution reaches every connected client.

## See how they fit together <a id="architecture"/>

The integration follows a straightforward pattern:

1. **Client** sends a message via HTTP to your server.
2. **Server** invokes the LangGraph agent and receives a stream of messages as it executes through the graph.
3. **Server** pipes each output from the graph stream into an Ably channel.
4. **AI Transport** delivers tokens to every subscribed client with ordering, persistence, and recovery.

The HTTP response is immediate. The agent's output stream is decoupled from the HTTP request and flows through Ably instead. This means a client can disconnect and reconnect while the agent is still executing without losing any output.

<Code>
```javascript
// Server: pipe LangGraph stream through Ably channel
const stream = await graph.stream(
  { messages },
  { streamMode: 'messages' },
);

for await (const [message, metadata] of stream) {
  if (message.content) {
    await channel.publish('token', message.content);
  }
}
```
</Code>

## Use the core transport <a id="core-transport"/>

There is no dedicated LangGraph codec in AI Transport yet. You work with the core transport directly, publishing messages and tokens to the Ably channel yourself.

This gives you full control over what gets published and how. You decide the message format, the event names, and the structure. The core transport handles turn lifecycle, cancellation, and multi-client sync.

<Code>
```javascript
// Create the server transport with the core encoder
const transport = createServerTransport({ channel, codec });

// Start a turn and stream
const turn = transport.newTurn({ turnId, clientId });
await turn.start();
```
</Code>

## Leverage the graph architecture <a id="graph-architecture"/>

LangGraph's design pairs naturally with AI Transport's durable sessions. Several patterns emerge from this combination.

### Durable from model to client <a id="durable-end-to-end"/>

LangGraph checkpoints persist the agent's internal state. AI Transport sessions persist the delivery state. Together, you get durability from model to client. If the agent crashes, LangGraph resumes from its last checkpoint. If the client disconnects, AI Transport replays missed tokens on reconnect. Neither layer needs to know about the other.

### Multi-agent visibility <a id="multi-agent-visibility"/>

In a multi-agent LangGraph setup, subagents can publish directly to the session channel. Each agent node in the graph can emit status updates, intermediate results, or tokens. Clients see every agent's output in real time through a single channel, with ordering preserved.

<Code>
```javascript
// Each node in the graph can publish to the channel
const graph = new StateGraph(MessagesAnnotation)
  .addNode('researcher', async (state) => {
    await channel.publish('status', 'Researching...');
    const result = await researchModel.invoke(state.messages);
    return { messages: [result] };
  })
  .addNode('writer', async (state) => {
    await channel.publish('status', 'Writing response...');
    const result = await writerModel.invoke(state.messages);
    return { messages: [result] };
  })
  .addEdge('__start__', 'researcher')
  .addEdge('researcher', 'writer')
  .compile();
```
</Code>

### Agent crash recovery <a id="crash-recovery"/>

LangGraph's checkpointing and AI Transport's session persistence work independently but complement each other. When an agent process restarts, LangGraph picks up from the last checkpoint and the new process attaches to the same Ably channel. Clients see a brief pause, not a failure.

## Discover what this unlocks <a id="what-this-unlocks"/>

### Problems that go away <a id="problems-solved"/>

LangGraph handles agent orchestration well. It is not built to solve transport-level problems. With AI Transport, these issues are resolved:

- Agent output streams that die on disconnect. AI Transport resumes from where you left off, automatically.
- No way for multiple clients to observe the same agent run. The session persists across any surface.
- No visibility into which graph node is currently executing. Status messages on the channel give real-time progress.
- Long-running agent tasks lost on mobile network drops. Ordering and persistence mean nothing is lost.

### New capabilities <a id="new-capabilities"/>

Things your app can do that were not possible with LangGraph alone:

- Observe the same agent run on phone, laptop, and tablet, all in sync.
- Regenerate, edit, and navigate conversation branches. Each branch is a fork in the conversation tree.
- Cancel, interrupt, and steer agents mid-execution through bidirectional control.
- Approval gates reach the user on any device, even after reconnecting. LangGraph's interrupt pattern combined with AI Transport's durable delivery means approvals are never missed.
- Multiple agents or requests stream simultaneously, each with independent cancel handles.
