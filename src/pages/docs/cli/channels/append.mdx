<Code>
```shell
ably channels append <channel> <serial> <message> [flags]
```
</Code>

Use the `ably channels append` command to append data to a message on an Ably Pub/Sub channel.

## Arguments

Pass the following arguments to the `ably channels append` command:

### `<channel>` <a id="channel"/>

The name of the channel containing the message.

### `<serial>` <a id="serial"/>

The serial of the message to append data to.

### `<message>` <a id="message"/>

The data to append. This can be plain text or JSON.

## Flags

You can optionally pass the following flags to the `ably channels append` command:

### `--description` <a id="description"/>

A description for the appended data.

### `--encoding | -e` <a id="encoding"/>

The encoding of the message, such as `json/utf-8`.

### `--name | -n` <a id="name"/>

The event name for the appended message.

### `--client-id` <a id="client-id"/>

A client ID to use when appending the message.

## Global flags

The following [global flags](/docs/cli#global-flags) are also available:

* [`--json`](/docs/cli#json)
* [`--pretty-json`](/docs/cli#pretty-json)
* [`--verbose`](/docs/cli#verbose)

## Examples

Append data to a message:

<Code>
```shell
ably channels append my-channel "01a2b3c4d5@1234567890000-0" '{"extra": "data"}'
```
</Code>

Append with an event name:

<Code>
```shell
ably channels append my-channel "01a2b3c4d5@1234567890000-0" "additional info" --name update
```
</Code>

Append with a description:

<Code>
```shell
ably channels append my-channel "01a2b3c4d5@1234567890000-0" '{"note": "correction"}' --description "Added correction"
```
</Code>
