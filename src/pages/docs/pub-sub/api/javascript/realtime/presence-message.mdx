A `PresenceMessage` represents an individual presence event sent to or received from Ably. Presence messages are delivered through a channel's [`presence.subscribe()`](/docs/pub-sub/api/javascript/realtime/realtime-presence#subscribe) method, or retrieved via [`presence.get()`](/docs/pub-sub/api/javascript/realtime/realtime-presence#get) and [`presence.history()`](/docs/pub-sub/api/javascript/realtime/realtime-presence#history).

<Code>
```javascript
await channel.presence.subscribe((presenceMessage) => {
  console.log(presenceMessage.action, presenceMessage.clientId);
});
```
</Code>

## Properties

The `PresenceMessage` type has the following properties:

<Table id='PresenceMessageProperties'>

| Property | Description | Type |
| --- | --- | --- |
| action | The type of [presence action](/docs/presence-occupancy/presence#trigger-events) this message represents. | <Table id='PresenceAction'/> |
| clientId | The client ID of the publisher of this presence message. | String |
| connectionId | The connection ID of the publisher of this presence message. | String |
| data | The presence data payload, if provided. | String, JSON Object, or Buffer |
| encoding | This will typically be empty as all presence messages received from Ably are automatically decoded client-side using this value. However, if the encoding cannot be processed, this attribute will contain the remaining transformations not applied to the `data` payload. | String |
| extras | Metadata and/or ancillary payloads, if provided. The only currently valid payloads for `extras` are the [`push`](/docs/push/publish#sub-channels), [`ref`](/docs/messages#interactions) and [`privileged`](/docs/platform/integrations/skip-integrations) objects. | JSON Object |
| id | A unique ID assigned to each `PresenceMessage` by Ably. | String |
| timestamp | Timestamp when the presence message was first received by Ably, as milliseconds since the epoch. | Integer |

</Table>

<Table id='PresenceAction' hidden>

| Value | Description |
| --- | --- |
| absent | Reserved for internal use. This is an internal enum value and is not delivered to subscribers as a presence event. |
| present | A member is present at the time of channel attach. Synthesized for each existing member when a client first subscribes. |
| enter | A new member entered the presence set. |
| leave | A member left the presence set, either explicitly or implicitly (e.g. due to disconnection). |
| update | A present member updated their data. |

</Table>

## Create a PresenceMessage from an encoded object <a id="from-encoded"/>

<MethodSignature>{`PresenceMessage.fromEncoded(encodedMsg: Object, channelOptions?: ChannelOptions): Promise<PresenceMessage>`}</MethodSignature>

A static factory method to create a `PresenceMessage` from a deserialized `PresenceMessage`-like object encoded using Ably's wire protocol. The returned promise resolves with the decoded presence message.

<Code>
```javascript
const presenceMessage = await Ably.Realtime.PresenceMessage.fromEncoded(encodedMsg);
```
</Code>

### Parameters <a id="from-encoded-params"/>

The `PresenceMessage.fromEncoded()` method takes the following parameters:

<Table id='FromEncodedParameters'>

| Parameter | Required | Description | Type |
| --- | --- | --- | --- |
| encodedMsg | Required | A `PresenceMessage`-like deserialized object. | Object |
| channelOptions | Optional | If you have an encrypted channel, use this to allow the SDK to decrypt the data. | [ChannelOptions](/docs/pub-sub/api/javascript/realtime/channels#get) |

</Table>

### Returns <a id="from-encoded-returns"/>

`Promise<PresenceMessage>`

Returns a promise that resolves with a `PresenceMessage` object decoded from the supplied object.

## Create a PresenceMessage from values <a id="from-values"/>

<MethodSignature>{`PresenceMessage.fromValues(values: Partial<PresenceMessage>): PresenceMessage`}</MethodSignature>

A static factory method that initialises a `PresenceMessage` from a `PresenceMessage`-like object. Only the `clientId`, `data`, and `extras` fields are used.

<Code>
```javascript
const presenceMessage = Ably.Realtime.PresenceMessage.fromValues({
  clientId: 'user-123',
  data: { status: 'available' }
});
```
</Code>

### Parameters <a id="from-values-params"/>

The `PresenceMessage.fromValues()` method takes the following parameters:

<Table id='FromValuesParameters'>

| Parameter | Required | Description | Type |
| --- | --- | --- | --- |
| values | Required | A `PresenceMessage`-like object to initialise from. Only `clientId`, `data`, and `extras` are used. | Object |

</Table>

### Returns <a id="from-values-returns"/>

`PresenceMessage`

Returns a `PresenceMessage` object initialised from the supplied values.
